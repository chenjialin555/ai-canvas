# AI Canvas — 工程架构与源码索引

面向新成员与维护者：说明**当前**整体架构、主要数据流、目录约定，以及 **TypeScript / Python 源文件索引**（与仓库一致；若有出入以代码为准）。

---

## 1. 项目是什么

**AI Canvas** 是基于 **React + Konva + Zustand** 的无限画布编辑器，配套 **Python FastAPI** 后端。支持多页面画布、矢量/位图元素、图层与小地图、单步 AI 生图（文生图 / 图生图 / 局部重绘），以及**画布内 AI 工作流节点**（端口连线、单节点远端执行、结果写回画布）。可通过 **Vite** 在浏览器运行，也可通过 **Electron** 打桌面包。

---

## 2. 技术栈与仓库边界

| 区域 | 技术 |
|------|------|
| 前端 | React 19、TypeScript、Vite、react-konva、Zustand、Immer（store 部分逻辑） |
| 桌面 | Electron（`electron/main.cjs`、`electron/preload.cjs`） |
| 后端 | Python 3.10+、FastAPI、httpx、Pydantic；模型调用抽象为 `providers` |
| 存储 | 阿里云 OSS（`backend/app/storage/oss.py`） |
| 持久化（客户端） | `localStorage` 自动保存 + JSON 工程 / json 工作流导入导出 |

**本索引不展开**：`node_modules/`、`dist/`、`release/`、`backend/logs/`、`package-lock.json`。

## 3. 当前前端目录总览

```text
src/
  main.tsx                 # 入口：样式、initUiTheme、initPlatform、挂载 App
  app/                     # 组装：App、shell/*、hooks/*
  features/                # 按业务域拆分
    editor/                # Zustand、命令、持久化、工程导出、图片滤镜、OSS 持久化
    canvas/                # Konva 画布、元素、交互、StageCanvas / MiniMap / FloatingToolbar
    workflow/              # 工作流 model、节点 UI、Comfy json 互导、runner、结果落画布
    ai-generation/         # 单步生图 Modal、准备位图、API、错误归一
    image-tools/           # 裁剪 / 蒙版 / 调色(adjust) / 全景
    library/               # 素材库面板
    ai-chat/               # AI 对话侧栏（复用生图 API）
    settings/              # 外观与快捷工具条设置
  shared/                  # 与单一业务弱耦合的基建
    api/                   # client、aiClient、uploadImageUrl
    platform/              # Web / Desktop API Base
    ui/                    # 通用 UI（如 ContextMenu）
    lib/                   # 主题、布局、调试、OSS 解析、yieldToMain 等
    hooks/                 # 跨模块 hooks
    types/                 # 全局 .d.ts 等
  styles/                  # 全局 CSS
```

**路径别名**：`tsconfig.app.json` 与 `vite.config.ts` 配置 `@/*` → `src/*`。业务代码推荐 `import { … } from "@/features/editor"` 等。

**各域统一出口**：`src/features/<name>/index.ts` 聚合对外 API。

```text
┌─────────────────────────────────────────────────────────────┐
│  app/：App.tsx + shell（顶栏 / 侧栏 / 画布区 / 弹窗）+ hooks   │
└───────────────────────────┬─────────────────────────────────┘
                            │ 组合、少写厚业务逻辑
┌───────────────────────────▼─────────────────────────────────┐
│  features/canvas：Konva Stage、Layer、元素节点、交互、浮动工具条 │
└───────────────────────────┬─────────────────────────────────┘
                            │ 订阅 / 派发
┌───────────────────────────▼─────────────────────────────────┐
│  features/editor：useEditorStore（多 slice）、commands、持久化 │
└───────┬───────────────────────────────┬─────────────────────┘
        │ localStorage                   │ fetch → /api/*
        ▼                               ▼
┌───────────────┐               ┌───────────────────────────────┐
│ editor 持久化  │               │  Vite 代理 → FastAPI /api     │
└───────────────┘               │  modules: generation /        │
                                │  upload / workflow / models   │
                                └───────────────┬───────────────┘
                                                │
                        ┌───────────────────────┼───────────────────────┐
                        ▼                       ▼                       ▼
                 ┌─────────────┐         ┌─────────────┐         ┌─────────────┐
                 │ providers   │         │ executors   │         │ OSS         │
                 └─────────────┘         └─────────────┘         └─────────────┘
```

**职责简述**

| 区域 | 职责 |
|------|------|
| `app/` | 启动与布局组装；通过 `@/features/*` 引用业务模块。 |
| `features/canvas` | 画布表现与交互；`FloatingToolbar` 挂载裁剪/蒙版/调整等入口。 |
| `features/editor` | 文档模型、Zustand、元素命令、工程导出、**画布位图栅格化**（供 AI）。 |
| `features/workflow` | 节点注册表、连线、远端 `run-node`、**Comfy json 工作流**互导。 |
| `features/ai-generation` | 单步生图 UI；生成前调用 `prepareImageElementForApi`。 |
| `features/image-tools` | 全屏/标签式图片编辑：裁剪、蒙版、调色、全景。 |
| `shared/api` + `shared/platform` | 统一 API 根地址、`fetch`、**data URL → OSS**。 |
| `backend/app/modules/*` | 按业务垂直：`router` + `schemas` + `service`。 |
| `backend/app/providers` | 各厂商生图实现（原图 + 蒙版多为多图 URL）。 |
| `backend/app/executors` | 工作流节点在后端的执行入口。 |

## 5. 主要数据流

### 5.1 编辑器状态与画布

1. `useEditorStore`：`src/features/editor/store/index.ts`（含 `createWorkflowSlice`）。
2. `StageCanvas` → `CanvasPageContent` → `ElementNode` 按 id 订阅单元素，减少无关重绘。
3. 用户操作经 `features/canvas/hooks/*` 与 `executeElementCommand.ts` 写回 store，并可能进入 `historySlice`。

### 5.2 持久化与工程导入导出

1. `setupAutoPersist.ts` 节流写入 `localStorage`。
2. 顶栏 **导入/导出 json 工作流**：`useProjectImportExport` + `loadProjectJSON`；导入时 `normalizePagesImagesToUrls` 可将 base64 图上传 OSS。
3. 图片入画布：`elementSlice` 在 `addElement` / 替换图时 `scheduleImageSrcPersist` → `persistImageSrcToOss`。

### 5.3 单步 AI 生图（含蒙版 / 裁剪 / 调色）

1. UI：`AiGenerateModal.tsx`（分阶段提示：合成 → 上传 → 模型推理）。
2. 若选中图片：`prepareImageElementForApi`（`editor/export/`）
   - 有裁剪/滤镜/蒙版外框/蒙版笔画时：`renderImageElementToDataURL`（JPEG，最长边 ≤2048）→ 上传 OSS；
   - 无上述编辑且 `src` 已是远程 URL：直接用 `el.src`；
   - 蒙版：`exportImageMaskToDataURLAtSize` → **上传 OSS**（请求体不再塞巨大 base64）。
3. `generationService` → `generationApi` → **`POST /api/generate-image`**。
4. 后端 `normalize_request_images` 再次 `ensure_url`；`mode=inpaint` 时 `image` + `mask` 分开发给 Provider（如 Banana 的 `image_urls` 含两张图 + prompt 说明）。
5. 默认 **新图层** 在参考图右侧；有 `aiMask` 时不替换原图。

### 5.4 AI 对话侧栏

`AiChatPanel` 附件同样走 `prepareImageElementForApi`，逻辑与弹窗生图一致。

### 5.5 工作流单节点执行

1. `workflowSlice` → `serializeWorkflowInputsForApi`（async，图片端口用渲染 OSS URL）。
2. `workflowRunner` → **`POST /api/workflow/run-node`** → `workflowResultToCanvas`。

### 5.6 Comfy json 工作流（顶栏）

- **导出**：`exportComfyWorkflow`（画布图 → OSS，`planCanvasImageExports` 用 `prepareImageElementForApi`）。
- **导入**：`importComfyWorkflow` → 写入当前页 `aiNodes` / `edges`（非 ComfyUI 原生 `nodes/links` 格式）。

### 5.7 API Base（Web / Desktop）

`initPlatform()` → `setApiBaseUrl`；Web 用 Vite `/api` 代理；Desktop 用 preload 注入地址。

## 7. 维护约定（依赖方向）

1. **`app` → `features` / `shared`；`features` 不得依赖 `app`。**
2. **`features` → `shared`；跨 feature 以 `editor` store 为枢纽**，避免环状依赖。
3. **对外 import 优先 `@/features/<域>`**。
4. **后端新 HTTP**：`modules/<域>/` 下 `router` / `schemas` / `service`。
