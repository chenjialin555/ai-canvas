# 3. Vue 和 React 最大区别

你之前学过 Vue，所以先用 Vue 对比 React。

---

## 3.1 Vue 文件一般这样写

```vue
<template>
  <div class="box">
    {{ title }}
  </div>
</template>

<script setup>
const title = '你好'
</script>

<style scoped>
.box {
  color: red;
}
</style>
```

Vue 把一个组件分成：

```text
template：HTML
script：JS
style：CSS
```

---

## 3.2 React 文件一般这样写

```tsx
export function Demo() {
  const title = "你好";

  return (
    <div className="box">
      {title}
    </div>
  );
}
```

React 没有 `<template>`。

React 的页面结构写在：

```tsx
return (
  ...
)
```

里面。

这部分叫：

```text
JSX / TSX
```

你可以先把它当成：

```text
写在 JS 里的 HTML
```

| Vue | React |
|---|---|

# 6. AppShell 是什么？

你的 `AppShell.tsx`：

```tsx
import type { ReactNode } from "react";

type AppShellProps = {
  onRootMouseDown: () => void;
  topBar: ReactNode;
  leftSidebar: ReactNode;
  canvasArea: ReactNode;
  rightSidebar: ReactNode;
  overlays: ReactNode;
};

export function AppShell(props: AppShellProps) {
  return (
    <div className="figma-app" onMouseDown={props.onRootMouseDown}>
      {props.topBar}
      <main className="figma-main">
        {props.leftSidebar}
        {props.canvasArea}
        {props.rightSidebar}
      </main>
      {props.overlays}
    </div>
  );
}
```

它其实非常简单。

它的作用是：

```text
摆页面布局
```

它不管业务。

它只负责：

```text
顶部放 topBar
中间放 leftSidebar + canvasArea + rightSidebar
最后放 overlays
```

页面结构大概是：

```text
figma-app
├── topBar
├── figma-main
│   ├── leftSidebar
│   ├── canvasArea
│   └── rightSidebar
└── overlays
```

---

## 6.1 ReactNode 是什么？

```ts
topBar: ReactNode;
```

`ReactNode` 可以理解成：

```text
任何可以显示在页面上的东西
```

比如：

```tsx
<TopBar />
<div>hello</div>
"文字"
null
```

所以：

```tsx
<AppShell topBar={<TopBar />} />
```

意思是：

```text
把 TopBar 这个组件传给 AppShell，让它显示在顶部。
```

这有点像 Vue 的插槽。

Vue 里可能写：

```vue
<AppShell>
  <template #topBar>
    <TopBar />
  </template>
</AppShell>
```

React 这里是：

```tsx
<AppShell topBar={<TopBar />} />
```

# 8. 这个项目里的状态分两类

你必须先理解这个。

## 8.1 项目真实数据：放在 store

比如：

```text

画布元素
选中的元素
AI 节点
连线

```

这些是项目本身的数据。

放在：


也就是 Zustand Store。

类似 Vue 的 Pinia。

比如：

```text

右侧栏当前 tab
```

## 8.3 举例

| 状态 | 放哪里 | 为什么 |
|---|---|---|
| 当前画布有哪些图片 | store | 这是项目内容 |

| AI 弹窗开没开 | modals | 临时界面状态 |
| 右键菜单 x/y | modals | 不需要保存到项目 |
| toast 提示文字 | modals | 临时提示 |
| 页面 zoom/pan | store | 画布视图状态 |

# 9. 什么是 Zustand？

你可以把 Zustand 理解成：

```text
React 里的 Pinia
```

Vue / Pinia 里可能是：

```ts
const store = useEditorStore()
store.selectedIds
store.updateElement(id, data)
```

React / Zustand 里是：

```tsx
const selectedIds = useEditorStore((s) => s.selectedIds);
const updateElement = useEditorStore((s) => s.updateElement);
```

---

## 9.1 为什么写成这样？

```tsx
const selectedIds = useEditorStore((s) => s.selectedIds);
```

意思是：

```text
我只关心 store 里的 selectedIds。
selectedIds 变化时，这个组件才需要重新渲染。
```

不要乱写：

```tsx
const store = useEditorStore();
```

因为这可能表示：

```text
整个 store 任何东西变化，这个组件都可能重新渲染。
```

对画布项目来说，这可能导致卡顿。

# 10. 什么是 Konva？

Konva 是这个项目最重要但也最容易陌生的部分。

你之前学 HTML，页面是：

```html
<div>
  <img />
  <p>Hello</p>
</div>
```

但画布不是普通 HTML。

Konva 是：

```tsx
<Stage>
  <Layer>
    <Rect />
    <Text />
    <Image />
  </Layer>
</Stage>
```

你可以理解成：

```text
Stage = 整个画布
Layer = 图层
Rect = 矩形
Text = 文字
Image = 图片
Group = 一组图形
Line = 线条
```

---

## 10.1 普通 HTML 和 Konva 对比

| 普通网页 | Konva |
|---|---|
| `<div>` | `<Group>` |
| `<img>` | `<Image>` |
| `<p>` / `<span>` | `<Text>` |
| CSS background | `fill` |
| border | `stroke` |
| left/top | `x/y` |
| transform | `scale/rotation` |
| click | `onClick` |
| drag | `draggable` |

---

## 10.2 画布内容不是 HTML

画布里的图片、矩形、文字，不是浏览器 DOM 里的：

```html
<img>
<div>
<p>
```

它们是画在 `<canvas>` 上的。

所以你在浏览器 Elements 面板里可能看不到每一个图片或矩形。

这是正常的。

# 12. 一个元素是怎么显示出来的？

这是你必须理解的一条线。

假设 store 里有一个矩形：

```ts
{
  id: "rect1",
  type: "rect",
  x: 100,
  y: 100,
  width: 200,
  height: 100,
  fill: "red"
}
```

流程是：

```text
store 里有 element 数据
↓
ContentLayer 遍历 elements
↓
ElementNode 看 type
↓
type 是 rect
↓
交给 RectElementNode
↓
RectElementNode 返回 Konva <Rect />
↓
画布上出现矩形
```

也就是：

```text
数据 → React 组件 → Konva 图形 → 画布显示
```

# 13. 用户拖动元素时发生什么？

比如拖动一张图片：

```text
用户拖动图片
↓
Konva 自己让图片跟着鼠标动
↓
拖动结束 onDragEnd
↓
调用 updateElement(id, { x, y })
↓
Zustand Store 更新元素位置
↓
React / Konva 根据新数据重新渲染
↓
位置保存下来
```

注意：

```text
拖动中最好不要一直写 store
拖动结束再写一次
```

这是画布性能的重要原则。

# 14. 什么是 zoom 和 pan？

画布有两个重要状态：

```text
zoom：缩放
pan：平移
```

---

## 14.1 zoom

表示画布放大多少。

比如：

```text
zoom = 1     原始大小
zoom = 2     放大两倍
zoom = 0.5   缩小一半
```

---

## 14.2 pan

表示画布整体移动了多少。

比如：

```ts
pan = { x: 100, y: 50 }
```

表示画布往右移动 100，往下移动 50。

---

## 14.3 世界坐标和屏幕坐标

这是画布项目里最容易混的概念。

### 世界坐标

元素自己的位置。

比如：

```ts
element.x = 300
element.y = 200
```

这个位置不因为你缩放画布而改变。

---

### 屏幕坐标

鼠标在浏览器窗口里的位置。

比如：

```text
clientX = 800
clientY = 400
```

---

### 为什么要转换？

因为用户鼠标点的是屏幕坐标，但元素存在世界坐标。

所以需要：

```text
screenToWorld
worldToScreen
```

# 23. 这个项目的核心数据流

你要反复记这条线：

```text
用户操作
↓
React / Konva 事件
↓
调用 store action
↓
修改 Zustand store
↓
React / Konva 根据新数据重新显示
```

例如拖动元素：

```text
拖动图片
↓

↓
updateElement(id, { x, y })
↓
store 里的 element 位置变了
↓
画布显示新位置
```

```tsx
import { useState } from "react";

export function LearnReact() {

  const [visible, setVisible] = useState(true);

  return (

      <h1>React 学习</h1>

}
```

```tsx
import { StrictMode } from "react";
```
