## 2. 四个 feature 核心

### 2.1 `features/editor` — 大脑（数据）

```text
features/editor/
  store/           # Zustand：pages、elements、viewport、workflow…
  commands/        # executeElementCommand
  export/          # 工程 JSON、栅格化、AI 用图准备
  image-filter/    # 调色参数
  images/          # 入画布后 OSS 持久化
  quick-tools/     # 浮动工具条配置
  types.ts
```

**数据在 store，不在 Konva。** 矩形的位置在 `elementSlice`，Konva 只负责画出来。

关键文件：`features/editor/store/index.ts`。

### 2.2 `features/canvas` — 显示器 + 交互

```text
features/canvas/
  components/      # StageCanvas、FloatingToolbar、MiniMap
  layers/          # Background / Content / Interaction
  elements/        # ImageElementNode、Rect、Text…
  interactions/    # Transformer、框选、工作流临时连线
  hooks/           # 平移缩放、快捷键、视口同步
```

入口：`StageCanvas.tsx` → `CanvasPageContent` → 各 Layer。

图片相关：

- `ImageElementNode.tsx`：裁剪变换 + `useFilteredImageSource` 调色 + `AIMaskOverlay` 蒙版预览
- `FloatingToolbar.tsx`：裁剪/调整/蒙版/连线等；调整用 `ImageAdjustPanel`（portal）

### 2.3 `features/workflow` — 画布内 AI 节点

```text
features/workflow/
  model/           # nodeRegistry、各节点定义
  components/      # WorkflowNodeView、NodePicker
  store/           # workflowSlice（合入 editor store）
  utils/           # unifiedGraph、runPayload（async 序列化）
  comfy/           # 导出/导入 json 工作流（非 ComfyUI 原生格式）
  services/        # workflowRunner、workflowResultToCanvas
  api/             # workflowApi
```

运行节点：

```text
点击运行
  → workflowSlice / workflowRunner
  → POST /api/workflow/run-node
  → backend executors
  → workflowResultToCanvas 写回画布
```

### 2.4 其它 features（简表）

| 目录 | 用途 |
|------|------|
| `ai-generation` | **单步弹窗**生图（非工作流节点） |
| `image-tools` | 全屏图片编辑：裁剪、蒙版、全景；`adjust` 在浮动条 |
| `ai-chat` | 右侧「AI对话」；附件同样走生图 API |
| `library` | 素材库 |
| `settings` | 外观、快捷工具条 |

## 3. 单步 AI 生图（重点）

### 3.1 和 workflow 的区别

| | 单步生图 | 工作流 |
|---|---------|--------|
| 入口 | 顶栏 / 右键 / 弹窗 | 画布上 AI 节点 |
| 目录 | `features/ai-generation` | `features/workflow` |
| API | `POST /api/generate-image` | `POST /api/workflow/run-node` |

### 3.2 完整流程（当前实现）

```text
AiGenerateModal 点「生成」
  ↓
generationService.generateImageFromModal
  ↓
【若有选中图片】prepareImageElementForApi
      ├─ 需要时：renderImageElementToDataURL（裁剪+滤镜，JPEG，最长边≤2048）
      ├─ 上传合成图 → OSS → imageUrl
      └─ 有 aiMask：exportImageMaskToDataURLAtSize → 上传 OSS → maskUrl
  ↓
buildModalGenerateImagePayload（mode: generate | image-to-image | inpaint）
  ↓
postGenerateImage → shared/api/aiClient
  ↓
POST /api/generate-image
  ↓
backend generation/service.normalize_request_images（再次 ensure_url）
  ↓
providers（如 Banana：image_urls = [原图, 蒙版图] + prompt 说明）
  ↓
返回 url → addElement 新图层（默认在参考图右侧，不覆盖原图蒙版）
```

### 3.3 蒙版为什么「图里看不见却有效」

- **`image` 字段**：发给模型的是**干净底图**（裁剪+调色后的合成），**不含红色涂抹**。
- **`mask` 字段**：**另一张图**（笔画栅格化的 PNG，已上传 OSS）。
- 多模态模型同时看两张图 + 你的 prompt（如「只改蒙版区域」）。

画布上应能看到红色半透明叠加（`AIMaskOverlay`）；若看不见，多半是预览层问题，**不影响**已保存的 `aiMask` 参与 API。

### 3.4 为什么会「卡」半分钟

| 阶段 | 原因 |
|------|------|
| 准备（数秒） | 大图 Canvas 编码、上传 OSS；已做分辨率上限与 JPEG |
| 模型（20～60s+） | 网关推理，弹窗会显示「模型推理中」 |

弹窗内 `onPhase`：`render` → `upload` → `request` → `model`。

## 4. 图片：裁剪、调色、蒙版

| 能力 | 入口 | 数据字段 | 发给 AI |
|------|------|----------|---------|
| 裁剪 | 浮动条 / 右键 → `ImageEditorModal` crop | `cropOffset*`、`cropScale`… | 栅格化进 `imageUrl` |
| 调色 | 浮动条「调整」→ `ImageAdjustPanel` | `filter` | 栅格化进 `imageUrl` |
| AI 蒙版 | 浮动条「蒙版」→ `MaskEditorPanel` | `aiMask.strokes` | 单独 `mask` URL |
| 外框形状 | 裁剪编辑器 | `maskShape`、`cornerRadius` | 栅格化 clip |

无裁剪/滤镜/蒙版且 `src` 已是 OSS 时，生图可**跳过栅格化**，直接用原 `src`。

## 5. 后端（简图）

```text
POST /api/generate-image
  → modules/generation/router.py
  → service.normalize_request_images（upload ensure_url）
  → providers/registry → banana.py / doubao.py / …

POST /api/upload-image-url
  → modules/upload（data URL → OSS）

POST /api/workflow/run-node
  → modules/workflow → executors/registry
```

`providers/payloads.py`：各厂商如何组 prompt、是否把 `mask` 放进 `image_urls` 等。

## 8. 易混概念

### editor vs canvas

```text
editor = 数据（store）
canvas  = 显示 + 鼠标键盘
```

### ai-generation vs workflow

```text
ai-generation = 单次弹窗 / 侧栏对话生图
workflow      = 节点图、可连线、可导出 json 工作流
```

### shared/api vs feature/api

```text
shared/api/aiClient.ts     → 怎么 POST JSON
features/ai-generation/api → 生图路径与解析
```

### 工程 JSON vs json 工作流

- **工程 JSON**：`pages` + `elements` + `aiNodes`，`loadProjectJSON` 整页加载。
- **json 工作流**：经 `exportComfyWorkflow` 导出、可含 Comfy 桥接节点；图片字段为 OSS URL。
