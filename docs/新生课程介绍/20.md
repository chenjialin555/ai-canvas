# 第二十课：`WorkflowNodeView` — 画布上「AI 节点卡片」怎么读

这一课面向 **单文件太长不敢下口** 的你：  
文件 **`src/components/workflow/WorkflowNodeView.tsx`**（行数会随版本变）是 **Konva 上的工作流节点 UI**，与第九课的 **`runWorkflowNode`** 强相关。

> 系列目录见 [README.md](./README.md)。上一课：[19.md](./19.md)。下一课：[21.md](./21.md)。

---

# 1. 它在树里的位置

**`CanvasPageContent`**（第十五课）对每个 AI 节点 id 渲染：

```tsx
<WorkflowNodeView key={id} nodeId={id} />
```

所以它 **在 Konva `Layer` 里**，和普通 `ElementNode` 是兄弟（顺序上端口叠层与节点视图有架构上的前后关系，见 `PROJECT_ARCHITECTURE`）。

---

# 2. 数据从哪读？

组件用 **`nodeId`** 从 store 里 **选出那一个 `WorkflowNode`**：

```text
useEditorStore((s) => page.aiNodes.find(n => n.id === nodeId))
```

另有 **`getWorkflowNodeDefinition(node.type)`**（**`src/workflow/nodeRegistry.ts`**）：决定 **端口列表、参数 schema、是否有运行条、executor 类型** 等。

人话：

```text
node   = 这一实例的数据（位置、状态、params、outputs）
def    = 这一类型的「说明书」
```

---

# 3. 画面上大致分几块？

从 JSX 结构理解即可（不必背每一行）：

```text
Group（整卡 draggable）— x/y 来自 node
  ├ 阴影 / 圆角底 Rect
  ├ 顶栏状态色（running / selected / idle）
  ├ 标题、状态文案
  ├ 输入 / 输出端口 Circle（拖线起点、命中区）
  ├ 参数摘要行（来自 def.params）
  ├ 预览区 KonvaImage（若有 image url）
  └ 底部运行条：主按钮 + 「落画布」等
```

**缩放很小时**（`zoom` 低于阈值）会进入 **compact** 模式，隐藏部分正文，避免一坨糊在屏幕上。

---

# 4. 拖动节点时发生什么？

`Group` 上 **`onDragStart` / `onDragEnd`**：

```text
dragStart：手势历史开始 + 抑制浮动工具条（与 Stage 拖动一致）
dragEnd：把 e.target.x()/y() 写回 updateWorkflowNode(..., { history: false })
         再 gestureHistoryDragEnd() 合并撤销栈
```

与画布元素的 **手势合并历史** 同一套思想（第四、六课提过 `interactionGestureHistory`）。

---

# 5. 「运行」按钮连到哪里？

组件从 store 取出 **`runWorkflowNode`**、**`sendWorkflowResultToCanvas`** 等方法。  
点击主按钮时调用 **`runNode(node.id)`**（即 `runWorkflowNode`），具体分支见 **第九课**（`output-view` / `executor === "none"` / 远端 `run-node`）。

---

# 6. 端口与连线

你会看到 **`startWorkflowConnecting`**、**`completeWorkflowConnectingToInput`** 一类方法：  
对应 **从端口拖线**、**松手接到另一端口** 的 store 逻辑（第九课 `workflowConnecting`）。

注释里有一句值得记住：**不订阅拖线指针的每一帧坐标**，避免拖线时 **所有节点每个动画帧都重渲染** —— 这是性能意识，读代码时看到「少订阅」要知道作者在防什么。

---

# 7. 预览图 `useHtmlImage`

小 hook：**给定 URL → `new Image()` → onload 后塞进 state → 给 `<KonvaImage image={...} />`**。  
URL 来自 **`node.outputs`** 里约定 key（如 `result`）的 **image / mask** 输出。
