# 第十五课：从 `CanvasElement` 到屏幕 — `ElementNode` 与各类 `*ElementNode`

这一课回答：

```text
store 里只有 JSON 一样的数据（type、x、y、width…），
屏幕上那张图 / 那段字是谁画出来的？
```

> 系列目录见 [README.md](./README.md)。上一课：[14.md](./14.md)。下一课：[16.md](./16.md)。

---

# 1. 入口：`CanvasPageContent` 列出一批 id

在 **`src/canvas/components/CanvasPageContent.tsx`** 里（大意）：

```text
从 store 取出当前页「根级」元素 id 列表（没有 parentId 的那些）
对每个 id 渲染一个 <ElementNode elementId={id} />
```

同一层还会渲染 **图片端口叠层**、**每个 AI 节点的 `WorkflowNodeView`**（第九课）。  
你现在只盯 **`ElementNode`** 这条链即可。

---

# 2. 分发器：`ElementNode`（`src/canvas/elements/ElementNode.tsx`）

它做三件事：

1. **按 id 订阅** store 里那一个元素（避免整页变化都重画所有节点，注释里也有说明）。  
2. 若元素不存在或 `visible === false`，返回 **`null`**（不画）。  
3. 看 **`element.type`**，分支到不同组件：

```text
image → ImageElementNode
rect  → RectElementNode
text  → TextElementNode
arrow → ArrowElementNode
group → GroupElementNode
```

这就是第五课说的 **「判别联合类型」**：`type` 字段决定走哪条渲染路径。

---

# 3. 具体某一种：`ImageElementNode` 等

每个 `*ElementNode.tsx` 里通常是 **react-konva** 的 `Image` / `Rect` / `Text` / `Group` / `Line` 等，把 **`element.x, element.y, width, height, rotation`** 绑到 Konva 属性上。

**拖动结束、缩放结束** 时，会调 **`updateElement`**（或经命令 / 手势封装）把新几何写回 store —— 第七课的世界坐标 + 第六课的 store 在这里闭合。

---

# 4. `commonProps.ts` 是干什么的？

多个元素节点会共享 **拖拽、选中、变换** 等逻辑。项目里抽在 **`elements/commonProps.ts`**（及若干 hook）里，避免五个 `*ElementNode` 各复制一坨。

你读代码时：**先读 `ElementNode` 分支，再进对应 `*ElementNode`，遇到重复行为再跳 `commonProps`**。

---

# 5. 坐标：`canvas/utils/coordinates.ts`

指针在 Stage 上的位置和 **store 里的世界坐标** 往往不一致（有 **zoom / pan**）。  
换算、工具函数集中在 **`coordinates.ts`** 一类文件里，而不是散落在每个组件里。

人话：

```text
「鼠标在哪」和「元素在画布数学坐标里在哪」之间的换算，找 utils。
```

---

# 6. 和 `image-tools/` 的分工（防混）

| 区域 | 职责 |
|------|------|
| `canvas/elements/*` | **画布上** 常驻显示：图、字、框、箭头、组 |
| `image-tools/*` | **弹窗里** 的裁剪 / 蒙版 / 全景等编辑器，确认后写回 store |

第十二课：`openImageEditor` 打开的是 **工具侧**；画布侧只负责 **展示当前 `ImageElement` 状态**。

# 7. 练习

1. 打开 `ElementNode.tsx`，默写五个 `type` 分支各对应哪个文件名（可翻目录核对）。  
2. 在 `CanvasPageContent` 里找到 `rootElementIds` 是怎么从 `page.elements` 筛出来的。  
3. 全局搜 `updateElement`，看 `ImageElementNode`（或 common 拖拽）里是否在拖曳结束时调用。
