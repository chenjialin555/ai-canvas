# 第十八课：`main.tsx` — 应用从哪里启动、CSS 与主题谁先谁后

这一课只读 **一个文件**：`src/main.tsx`。  
读完你会知道：**浏览器第一次加载你的站点时，在 React 跑起来之前已经做了哪些事**。

> 系列目录见 [README.md](./README.md)。上一课：[17.md](./17.md)。下一课：[19.md](./19.md)。

---

# 1. 真实顺序（从上到下）

当前文件大意是：

```text
1. import React / createRoot / App
2. import initPlatform、initUiTheme
3. import 一串全局 CSS（editor、themes、皮肤…）
4. initUiTheme()
5. initPlatform()
6. createRoot(...).render(<StrictMode><App /></StrictMode>)
```

**关键点**：**先** `import` CSS（打包器会把样式注入页面），**再**调两个 `init*`，**最后**才 `render` 根组件。

---

# 2. 为什么要先 `import` CSS？

这些 `import "./styles/....css"` **没有赋给变量**，副作用是：**打包时把样式表打进产物**，浏览器执行到这段脚本时样式已可用。

人话：

```text
不要等到 App 里再写 <link>；入口文件统一拉主题和布局样式。
```

---

# 3. `initUiTheme()` 做什么？

实现见 **`src/lib/uiTheme.ts`**（本课只记行为）：

```text
从 localStorage 读出上次选的主题 id（读不到则用默认）
给 document.body 加上对应 class（并移除其它主题 class）
写回 localStorage，并 dispatch 一个自定义事件，通知 Konva 等非纯 CSS 区域刷新
```

注释写得很清楚：**在 React 挂载前调用，减少「先白屏再换肤」的闪烁**。

---

# 4. `initPlatform()` 做什么？

实现见 **`src/platform/index.ts`**：

```text
detectPlatform() — 若存在 window.desktopAPI 则视为桌面端，否则 Web
创建对应 PlatformAdapter
调用 setApiBaseUrl(adapter.getApiBaseUrl())
```

**下一课**会展开：为什么 Web 下 `getApiBaseUrl` 经常是 **空字符串**，以及这和 **`/api` 相对路径**（第八课）如何配合。

---

# 5. `StrictMode` 是什么？

```tsx
<StrictMode>
  <App />
</StrictMode>
```

开发环境下，React 可能 **故意双重调用** 部分生命周期 / effect，帮你发现副作用问题（第四课提过 StrictMode 与 `useEffect`）。  
生产构建行为以官方文档为准；你读日志时若看到「多打一次」，先想想是不是 StrictMode。

---

# 6. `createRoot` 与 `document.getElementById("root")`

```tsx
createRoot(document.getElementById("root")!).render(...)
```

`index.html` 里必须有 `<div id="root"></div>`。  
`!` 是 TypeScript 非空断言（第五课）：告诉编译器「这里一定有元素」。

# 7. 练习

1. 数一下 `main.tsx` 里共 `import` 了几个 CSS 文件。  
2. 打开 `uiTheme.ts`，找到 `STORAGE_KEY` 主题键名。  
3. 临时把 `initUiTheme()` 注释掉再 `npm run dev`，观察首屏是否更容易「闪主题」。

