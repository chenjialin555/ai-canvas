# 第十七课：统一图片编辑器 — `ImageEditorModal` 与三个 Tab

这一课讲 **`src/image-tools/ImageEditorModal.tsx`**：你在快捷条点「裁剪 / 蒙版 / 解析3D」时弹出的 **全屏蒙层 + Tab** 是怎么组织的。

> 系列目录见 [README.md](./README.md)。上一课：[16.md](./16.md)。下一课：[18.md](./18.md)。

---

# 1. 谁打开它？

**`useAppModals`** 里有 `imageEditor: { imageId, tool } | null`（第十二课）。  
`App` / `CanvasArea` 传下来的 **`onFloatingCrop` 等** 最终会 **`openImageEditor(id, "crop")`**。

`EditorModals` 里渲染：

```tsx
<ImageEditorModal
  open={!!modals.imageEditor}
  imageId={modals.imageEditor?.imageId ?? null}
  tool={modals.imageEditor?.tool ?? "crop"}
  onToolChange={...}
  onClose={modals.closeImageEditor}
/>
```

人话：

```text
open + imageId 有值 → 显示编辑器；关掉 → imageEditor 置 null。
```

---

# 2. 外壳做三件事

仍看 **`ImageEditorModal.tsx`**：

1. **`open && imageId`** 才渲染；否则 `return null`（第三课条件渲染）。  
2. **全屏遮罩** `modal-mask`：点 **遮罩本身**（`e.target === e.currentTarget`）关闭 —— 典型「点外面关弹窗」。  
3. **`useEffect` + Escape**：焦点不在输入框时按 Esc 关闭（第四课：注册/清理键盘监听）。

---

# 3. Tab 与三个子面板

头部用 **`IMAGE_EDITOR_TABS`**（`src/image-tools/types.ts`）渲染按钮，`tool === tab.id` 加 `active` 样式。

正文按 **`tool`** 分支：

```text
crop    → CropEditorPanel
mask    → MaskEditorPanel
parse3d → Panorama3DEditorPanel（等距柱状全景预览，偏只读）
```

每个 Panel 都接收 **`imageId`**：内部再去 **store** 里找对应 **`ImageElement`**，读 `src`、改 `crop*` / `aiMask` 等。

---

# 4. 和画布上的 `ImageElementNode` 分工

| 位置 | 职责 |
|------|------|
| `canvas/elements/ImageElementNode` | **画布上**日常展示与拖拽 |
| `image-tools/*Panel` | **弹窗里**做重编辑；确认后 **`updateElement` / `setImageAIMask`** 写回 store |

第十五课 + 本课连起来：**同一 `imageId`，两种 UI 壳子**。

---

# 5. 为什么裁剪、蒙版文件特别长？

`CropEditorPanel`、`MaskEditorPanel` 内含 **嵌套 Konva 子场景**、笔刷、与导出栅格化等逻辑（架构文档也有专节）。  
你第一遍读：**先找「保存 / 关闭时调用了哪个 store 方法」**，再决定要不要啃绘制细节。

# 6. 练习

1. 从 `ImageEditorModal` 跟到 `CropEditorPanel`，搜 `updateElement` 出现位置。  
2. 在 `types.ts` 里确认 `ImageEditorTool` 联合类型有哪三个字面量。  
3. 对比第十六课：浮动条是 **Portal + body**；本弹窗是 **普通 React 树里的 fixed 蒙层**（实现策略不同，都是 DOM 叠在画布上）。
