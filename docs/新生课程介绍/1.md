# 1. 先用你最熟的 HTML 来对比

在普通网页里，你通常这样写结构（`index.html`）：

```html
<div>
  <h1>你好</h1>
  <p>这是我的第一个页面</p>
</div>
```

再用 **JavaScript** 改文字、绑点击，例如：

```js
const title = document.querySelector("h1");
title.textContent = "你好，世界";

document.querySelector("button").addEventListener("click", () => {
  alert("点击了");
});
```

```text
HTML 文件里：写标签结构
.js 文件里：用 DOM API 找元素、改内容、监听事件
.css 文件里：写样式
```

React 做的事可以粗略理解成：**用 JS 函数「描述」一段界面结构**，由 React 帮你更新到真实 DOM 上（不必自己到处 `querySelector` 拼字符串）。

---

# 2. React 组件长什么样？

React 组件一般是一个函数。

```tsx
function Hello() {
  return (
    <div>
      <h1>你好</h1>
      <p>这是我的第一个 React 组件</p>
    </div>
  );
}
```

这就是一个 React 组件。

它和上面那段 **HTML** 长得几乎一样，只是：

```text
这段结构出现在 JS/TS 函数的 return 里，而且标签规则略像 HTML（叫 JSX，下一节讲）。
```

和「纯 HTML 文件」的常见区别：

```text
纯 HTML：结构多半写在 .html 里，和逻辑文件分开
React：结构（JSX）和逻辑写在同一个 .tsx 函数里，return 就是「这一屏要长什么样」
```

---

# 3. React 组件最简单的格式

一个 React 组件最基本长这样：

```tsx
function 组件名() {
  return (
    页面结构
  );
}
```

例如：

```tsx
function App() {
  return (
    <div>
      hello
    </div>
  );
}
```

---

# 4. JSX 是什么？

你看到的这部分：

```tsx
<div>
  <h1>你好</h1>
  <p>这是我的第一个 React 组件</p>
</div>
```

看起来很像 HTML。

但它不是普通 HTML。

它叫：

```text
JSX
```

如果文件是 TypeScript，就是：

```text
TSX
```

你可以先简单理解为：

```text
JSX = 写在 JavaScript 里面的“类似 HTML 的语法”
```

---

# 5. 普通 HTML 和 React JSX 对比

## 静态 HTML（浏览器直接认识）

```html
<div>
  <h1>你好</h1>
  <p>欢迎你</p>
</div>
```

## React（写在函数里，要经过编译）

```tsx
function Hello() {
  return (
    <div>
      <h1>你好</h1>
      <p>欢迎你</p>
    </div>
  );
}
```

你可以看到：**标签层级几乎一样**。  
差别是：React 把这段放在 **`.tsx` 文件的函数 `return (...)`** 里，并且最终会由 React 挂载到页面上的某个根节点（例如 `#root`）里。

---

# 6. React 组件必须返回一个整体

React 组件的 `return` 里通常要返回一个整体。

这样可以：

```tsx
function Hello() {
  return (
    <div>
      <h1>标题</h1>
      <p>内容</p>
    </div>
  );
}
```

不建议直接返回两个并列标签：

```tsx
function Hello() {
  return (
    <h1>标题</h1>
    <p>内容</p>
  );
}
```

这样会报错，因为没有一个共同的外层。

---

## 解决方法 1：包一个 div

```tsx
function Hello() {
  return (
    <div>
      <h1>标题</h1>
      <p>内容</p>
    </div>
  );
}
```

---

## 解决方法 2：用 Fragment

```tsx
function Hello() {
  return (
    <>
      <h1>标题</h1>
      <p>内容</p>
    </>
  );
}
```

这个：

```tsx
<>
</>
```

叫 Fragment。

意思是：

```text
我想返回多个元素，但不想额外加一个 div。
```

你的项目里也有：

```tsx
<>
  <ContextMenu />
  <EditorModals />
</>
```

就是这个意思。

---

# 7. JSX 里写变量

在**原生 JS** 里，你可能会先取节点再赋值：

```js
const title = "你好";
document.querySelector("#title").textContent = title;
```

React 里把「要显示的值」直接写在 JSX 里：

```tsx
function Hello() {
  const title = "你好";

  return (
    <h1>{title}</h1>
  );
}
```

注意：

```tsx
{title}
```

大括号的意思是：**这里插入一段 JavaScript 表达式的结果**（类似模板字符串里的 `${title}`，但是用在标签之间）。

---

# 8. JSX 里可以写表达式

```tsx
function Hello() {
  const name = "小明";

  return (
    <div>
      <p>你好，{name}</p>
      <p>1 + 1 = {1 + 1}</p>
      <p>{name.toUpperCase()}</p>
    </div>
  );
}
```

`{}` 里面可以写 JS 表达式。

例如：

```tsx
{name}
{1 + 1}
{name.toUpperCase()}
```

---

# 9. JSX 里不能直接写 if 语句

下面这样不行：

```tsx
return (
  <div>
    {if (show) {
      <p>显示</p>
    }}
  </div>
);
```

因为 `{}` 里要写表达式，不是语句。

---

## 常见写法 1：&&

```tsx
function Hello() {
  const show = true;

  return (
    <div>
      {show && <p>显示这段文字</p>}
    </div>
  );
}
```

意思是：

```text
show 为 true，就显示 p
show 为 false，就不显示
```

用原生 JS 做同一件事，你会写 `if` 再 `appendChild`，或改 `style.display`。  
JSX 里更常见的是上面这种 **表达式**，写在标签树里。

---

## 常见写法 2：三元表达式

```tsx
function Hello() {
  const loggedIn = false;

  return (
    <div>
      {loggedIn ? <p>已登录</p> : <p>未登录</p>}
    </div>
  );
}
```

用原生思路理解：相当于「根据 `loggedIn` 为真还是假，决定 innerHTML 放哪一段」，只是 JSX 用 **一行三元表达式** 写完了。

---

# 10. JSX 里渲染列表

原生 JS 里你可能会：

```js
const ul = document.querySelector("ul");
items.forEach((item) => {
  const li = document.createElement("li");
  li.textContent = item.name;
  ul.appendChild(li);
});
```

React 里：

```tsx
function UserList() {
  const items = [
    { id: "1", name: "小明" },
    { id: "2", name: "小红" },
  ];

  return (
    <ul>
      {items.map((item) => (
        <li key={item.id}>
          {item.name}
        </li>
      ))}
    </ul>
  );
}
```

React 用：

```tsx
items.map(...)
```

来循环。

---

## 为什么要 key？

```tsx
<li key={item.id}>
```

`key` 是 React 用来识别列表项的。

你可以先理解为：

```text
循环渲染列表时，每一项都要有一个唯一 key。
```

可以联想：你自己用 `for` 循环往 DOM 里加 `li` 时，也会用 `item.id` 区分每一项；React 用 `key` 来追踪「哪一项是谁」，列表变化时少出错。

---

# 11. JSX 里的 class 写法

**普通 HTML**：

```html
<div class="box"></div>
```

React：

```tsx
<div className="box"></div>
```

注意：

```text
React 里是 className，不是 class。
```

因为 `class` 是 JavaScript 的关键字。

---

# 12. JSX 里的 style 写法

HTML：

```html
<div style="color: red; font-size: 20px;"></div>
```

若用 **JavaScript 操作 style**，你会写对象：

```js
el.style.color = "red";
el.style.fontSize = "20px";
```

React：

```tsx
<div style={{ color: "red", fontSize: 20 }}>
  文字
</div>
```

注意是两个大括号：

```tsx
style={{ color: "red" }}
```

外层 `{}` 表示进入 JS 表达式。  
内层 `{}` 是 JS 对象。

---

# 13. JSX 里的事件

**原生 HTML + JS** 里你会写：

```html
<button id="btn">点击</button>
```

```js
document.getElementById("btn").addEventListener("click", handleClick);
```

React：

```tsx
<button onClick={handleClick}>点击</button>
```

完整例子：

```tsx
function ButtonDemo() {
  function handleClick() {
    alert("点击了");
  }

  return (
    <button onClick={handleClick}>
      点击
    </button>
  );
}
```

也可以直接写：

```tsx
<button onClick={() => alert("点击了")}>
  点击
</button>
```

注意 React 是：

```tsx
onClick
onMouseDown
onMouseMove
onChange
```

它们对应 DOM 里的 **`onclick` / `onmousedown` / `onchange`** 等，只是 React 用 **驼峰命名** 写在属性上。

---

# 14. 组件名必须大写

React 组件名要大写。

正确：

```tsx
function TopBar() {
  return <div>顶部栏</div>;
}
```

使用：

```tsx
<TopBar />
```

错误：

```tsx
function topBar() {
  return <div>顶部栏</div>;
}
```

因为小写标签 React 会当成普通 HTML 标签。

例如：

```tsx
<div />
<button />
<input />
```

这些是普通 HTML 标签。

而：

```tsx
<TopBar />
<AppShell />
<CanvasArea />
```

这些是 React 组件。

---

# 15. 组件可以嵌套组件

```tsx
function Header() {
  return <h1>标题</h1>;
}

function App() {
  return (
    <div>
      <Header />
      <p>内容</p>
    </div>
  );
}
```

这和 **HTML 里 `<div>` 里再套 `<header>`** 一样：大组件里套小组件，只是这里套的是你自己写的函数组件。


# 19. 对应你的项目：条件渲染

```tsx
{modals.toolbarToast && (
  <div className="toolbar-toast" role="status">
    {modals.toolbarToast}
  </div>
)}
```

意思是：

```text
如果 modals.toolbarToast 有内容，就显示提示条。
如果没有内容，就不显示。
```

用原生 JS 写，你会先 `if (!msg) return;` 再创建节点；JSX 里用 `&&` 把「有没有内容」和「要不要这一块标签」绑在一起。

# 20. 今天你只需要掌握这些

今天不要学太多。

你只要掌握：

```text
1. React 组件是函数
2. return (...) 里面是 JSX，长得像 HTML，但运行在 JS 里
3. JSX 里用 {变量} 显示数据
4. class 要写 className
5. 事件写 onClick / onMouseDown
6. 条件显示用 && 或三元表达式
7. 列表显示用 map
8. 多个元素可以用 <>...</>
```

# 27. 今天最后总结

React 最基础就是：

```tsx
function App() {
  const title = "你好";

  return (
    <div className="app">
      <h1>{title}</h1>
      <button onClick={() => alert("点击")}>
        按钮
      </button>

  );
}
```

你只要记住：

```text
function App() = 组件

{title} = 显示 JS 变量
className = class
onClick = 点击事件
```

这就是第一课的全部。
