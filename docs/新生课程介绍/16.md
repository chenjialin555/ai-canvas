# 第十六课：`FloatingToolbar` — Portal、屏幕坐标与何时隐藏

这一课讲 **画布旁边那一条浮动按钮**：它**不是**画在 Konva `Stage` 里的，而是 **DOM**，还要贴在 **选中元素的屏幕位置** 上。

> 系列目录见 [README.md](./README.md)。上一课：[15.md](./15.md)。下一课：[17.md](./17.md)。

---

# 1. 它挂在哪？

在 **`CanvasArea`** 里，`FloatingToolbar` 和 **`StageCanvas` 是兄弟**（都在 `section.canvas-area` 里），不是 `Stage` 的子节点。

人话：

```text
Konva 管画布里的图；浮动条是普通的 HTML 按钮条，叠在画布区域上方。
```

---

# 2. 为什么要 `createPortal`？

文件：`src/components/FloatingToolbar.tsx`。

末尾会把工具条 **`createPortal(..., document.body)`** 挂到 **`document.body`**。

常见原因：

```text
1. 避免被父级 overflow:hidden 裁切
2. 避免被某层 transform 影响 fixed 定位行为
3. z-index 更好控制，保证浮在整页最上层之一
```

你在浏览器 DevTools 里看 DOM：工具条节点往往出现在 **`body` 最底下**，而不是紧挨着 `Stage` 容器。

---

# 3. 位置怎么算？（世界 → Stage 内像素 → 屏幕 CSS 像素）

核心链条（变量名以源码为准）：

1. **选中元素**在 store 里是 **世界坐标** 下的 `x/y/width/height/rotation`（第七课）。  
2. 代码先算 **世界坐标系里的轴对齐包围盒 AABB**（旋转矩形四角转一圈取 min/max）。  
3. 用 **`worldToStageClient`**：把世界点乘 `zoom` 再加 `pan`，得到 **Stage 内容坐标系**里的点。  
4. 再加上 **`stage.container().getBoundingClientRect()`** 的偏移，得到 **视口 CSS 像素**下的 `left/top`，给 **`position: fixed`** 用。

人话：

```text
store 里是「数学画布」
浮动条要贴在「浏览器窗口」的某个像素上
中间必须经过 Stage 容器在屏幕上的矩形
```

---

# 4. 什么时候不显示？

`useLayoutEffect` 里有一组 **早退条件**，大意包括：

```text
没有可显示的快捷按钮
正在编辑文字（editingTextId）
框选中（marqueeSelecting）
floatingToolbarSuppressed 为 true（拖舞台 / 拖元素时由 store 置位）
选中有不可见元素
```

满足任一条，`pos` 置 `null`，工具条不出现。

---

# 5. 为什么用 `useLayoutEffect` + `requestAnimationFrame`（甚至两次）？

测量前要等 **Konva Transformer / 子节点** 把这一帧布局算完，否则 `getBoundingClientRect` 可能还是旧值或 0。  
`ResizeObserver` 监听 **Stage 容器尺寸变化**（窗口缩放、侧栏宽度变化）时重新测量。

你先记结论：

```text
「DOM 浮动层对齐 Konva」这类需求，几乎总要 rAF / ResizeObserver 之一。
```

---

# 6. 按钮从哪来？`quickTools` + store 配置

工具定义在 **`src/editor/quickTools.ts`**（`QUICK_TOOL_LIBRARY`、`getToolById` 等）。  
**白名单顺序**存在 **`useEditorStore` 的 `quickToolbarConfig`** 里（按单选类型 `image` / `text` / … 以及多选 `multi` 分档）。

设置 UI 在 **`QuickToolbarSettings`**（第十二课 `EditorModals` 里挂载）。

---

# 7. 点按钮之后谁干活？

`FloatingToolbar` 里不少操作直接 **`executeElementCommand`**（第十三课）；  
和 **裁剪 / 蒙版 / AI / 素材库** 相关的，则调用 **`CanvasArea` 经 `App` 传进来的 props**（例如 `onCrop` → `modals.openImageEditor(id, "crop")`）。

3. 在 store 里搜 `setFloatingToolbarSuppressed`，看谁在拖动画布时把它设为 `true`。
