# 第三课：useState 组件自己的状态

> 系列目录见 [README.md](./README.md)。

这一课我们只学一个东西：

```text
useState
```

你项目里的弹窗开关、右键菜单显示隐藏、侧栏 tab、toast 提示，本质上都和 `useState` 有关系。

你学完这一课，就能理解：

```tsx
const [open, setOpen] = useState(false);
```

以及为什么：

```tsx
setOpen(true)
```

页面会变化。

---

# 1. 先用原生 JS 想「状态」是什么

在**没有 React** 的页面里，计数器常写成：

```html
<p id="countLabel">点击次数：0</p>
<button id="btn" type="button">+1</button>
```

```js
let count = 0;

document.getElementById("btn").addEventListener("click", () => {
  count += 1;
  document.getElementById("countLabel").textContent = `点击次数：${count}`;
});
```

这里：

```text
count 是状态
count 变了，你要自己改 DOM，页面才会变
```

---

# 2. React 里的 useState

React 里对应写法是：

```tsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <button onClick={() => setCount(count + 1)}>
      点击次数：{count}
    </button>
  );
}
```

React 帮你做了两件事：**记住 `count`**，并在 **`setCount` 之后自动重算界面**（不必手写 `textContent`）。

---

## 人话解释

```tsx
const [count, setCount] = useState(0);
```

意思是：

```text
创建一个状态 count。
初始值是 0。
setCount 是修改 count 的函数。
```

---

# 3. useState 的固定格式

```tsx
const [状态名, 修改状态的函数] = useState(初始值);
```

例如：

```tsx
const [count, setCount] = useState(0);
const [open, setOpen] = useState(false);
const [name, setName] = useState("小明");
```

对应：

| 状态 | 初始值 | 修改函数 |
|---|---|---|
| `count` | `0` | `setCount` |
| `open` | `false` | `setOpen` |
| `name` | `"小明"` | `setName` |

---

# 4. 为什么不能直接改状态？

React 里不要这样：

```tsx
count = count + 1;
```

要这样：

```tsx
setCount(count + 1);
```

因为 React 需要通过 `setCount` 知道：

```text
状态变了，我要重新渲染页面。
```

对比原生写法：以前是你 **`count` 变了就自己更新 DOM**；  
React 里是你 **`setCount` 通知框架**，由框架决定怎么高效更新页面。

---

# 5. 最简单例子：计数器

```tsx
import { useState } from "react";

export function CounterDemo() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>当前数量：{count}</p>

      <button onClick={() => setCount(count + 1)}>
        +1
      </button>

      <button onClick={() => setCount(count - 1)}>
        -1
      </button>
    </div>
  );
}
```

流程：

```text
点击 +1
↓
执行 setCount(count + 1)
↓
count 改变
↓
组件重新执行
↓
页面显示新的 count
```

---

# 6. React 组件重新执行是什么意思？

这是 React 很重要的点。

当你调用：

```tsx
setCount(count + 1)
```

React 会重新执行这个组件函数：

```tsx
function CounterDemo() {
  ...
}
```

然后重新得到新的 JSX。

你可以简单理解为：

```text
状态变了 → 组件重新计算页面结构 → 页面更新
```

# 7. 布尔状态：打开 / 关闭

你项目里的弹窗就是这种。

```tsx
import { useState } from "react";

export function ModalDemo() {
  const [open, setOpen] = useState(false);

  return (
    <div>
      <button onClick={() => setOpen(true)}>
        打开弹窗
      </button>

      <button onClick={() => setOpen(false)}>
        关闭弹窗
      </button>

      {open && (
        <div className="modal">
          我是弹窗
        </div>
      )}
    </div>
  );
}
```

---

## 人话解释

```tsx
const [open, setOpen] = useState(false);
```

意思：

```text
open 表示弹窗是否打开。
初始是 false，不显示。
```

点击打开：

```tsx
setOpen(true)
```

open 变成 true。

然后：

```tsx
{open && <div>我是弹窗</div>}
```

会显示。

---

# 8. 和「原生 JS 控制显示/隐藏」对照

原生里你可能用 **`display` 或移除节点**：

```js
let open = false;
const box = document.getElementById("modal");

function render() {
  box.style.display = open ? "block" : "none";
}

document.getElementById("openBtn").addEventListener("click", () => {
  open = true;
  render();
});
```

React：

```tsx
const [open, setOpen] = useState(false);

<button onClick={() => setOpen(true)}>打开</button>

{open && <div>我是弹窗</div>}
```

对照（帮助记忆）：

| 原生思路 | React |
|---|---|
| 自己维护 `let open` | `const [open, setOpen] = useState(false)` |
| `open = true` 再手动改 DOM | `setOpen(true)`，由 React 更新 |
| `if (open) 显示 div` | `{open && <div />}` |

# 9. 切换状态

有时候你想点击一次显示，再点一次隐藏。

```tsx
export function ToggleDemo() {
  const [visible, setVisible] = useState(true);

  return (
    <div>
      <button onClick={() => setVisible(!visible)}>
        显示 / 隐藏
      </button>

      {visible && <p>这是一段文字</p>}
    </div>
  );
}
```

这里：

```tsx
setVisible(!visible)
```

意思是：

```text
如果 visible 是 true，就改成 false。
如果是 false，就改成 true。
```

# 10. 字符串状态：输入框

```tsx
import { useState } from "react";

export function InputDemo() {
  const [text, setText] = useState("");

  return (
    <div>
      <input
        value={text}
        onChange={(e) => setText(e.target.value)}
      />

      <p>你输入的是：{text}</p>
    </div>
  );
}
```

---

## 人话解释

```tsx
const [text, setText] = useState("");
```

表示：

```text
text 是输入框内容。
```

```tsx
value={text}
```

表示：

```text
输入框显示 text。
```

```tsx
onChange={(e) => setText(e.target.value)}
```

表示：

```text
用户输入时，把输入框最新内容存到 text。
```

---

# 11. React 输入框为什么要 value + onChange？

React 常见写法是“受控组件”。

```tsx
<input value={text} onChange={...} />
```

意思是：

```text
输入框显示什么，由 React 状态 text 决定。
用户输入后，onChange 修改 text。
text 改了，输入框显示也变。
```

流程：

```text
用户输入
↓
onChange
↓
setText
↓
text 更新
↓
input 显示新 text
```

# 12. 对象状态

有时候状态是一个对象。

比如右键菜单：

```tsx
const [menu, setMenu] = useState({
  visible: false,
  x: 0,
  y: 0,
});
```

显示右键菜单：

```tsx
setMenu({
  visible: true,
  x: 100,
  y: 200,
});
```

隐藏右键菜单：

```tsx
setMenu({
  ...menu,
  visible: false,
});
```

---

## `...menu` 是什么？

```tsx
setMenu({
  ...menu,
  visible: false,
});
```

意思是：

```text
复制原来的 menu 所有字段，
然后把 visible 改成 false。
```

如果原来：

```ts
menu = {
  visible: true,
  x: 100,
  y: 200
}
```

执行后变成：

```ts
{
  visible: false,
  x: 100,
  y: 200
}
```

---

# 13. 这和你的项目对应

你的项目里有：

```tsx
modals.setContextMenu((m) => ({ ...m, visible: false }));
```

现在你可以理解为：

```text
把右键菜单状态复制一份，
只把 visible 改成 false，
所以菜单隐藏。
```

# 14. 函数式更新

有时候你会看到：

```tsx
setCount((c) => c + 1);
```

这和：

```tsx
setCount(count + 1);
```

类似。

但在一些连续更新情况下，函数式写法更安全。

比如：

```tsx
setCount((c) => c + 1);
```

意思：

```text
拿到最新的 count，返回 count + 1。
```

---

## 对象也可以函数式更新

```tsx
setMenu((m) => ({
  ...m,
  visible: false,
}));
```

你的项目里这句：

```tsx
modals.setContextMenu((m) => ({ ...m, visible: false }));
```

就是函数式更新。

# 15. 数组状态

```tsx
export function ListDemo() {
  const [items, setItems] = useState(["苹果", "香蕉"]);

  return (
    <div>
      <button onClick={() => setItems([...items, "橘子"])}>
        添加橘子
      </button>

      {items.map((item) => (
        <p key={item}>{item}</p>
      ))}
    </div>
  );
}
```

---

## 不要直接 push

不要这样：

```tsx
items.push("橘子");
setItems(items);
```

推荐：

```tsx
setItems([...items, "橘子"]);
```

意思是：

```text
创建一个新数组。
```

React 更喜欢不可变更新。

# 16. useState 和 props 的区别

| 名称 | 来源 | 谁能改 |
|---|---|---|
| props | 父组件传来 | 子组件不能直接改 |
| state | 组件自己创建 | 组件自己通过 setX 改 |

---

## 例子

```tsx
function Child(props: { title: string }) {
  const [open, setOpen] = useState(false);

  return (
    <div>
      <h1>{props.title}</h1>

      <button onClick={() => setOpen(true)}>
        打开
      </button>

      {open && <p>打开了</p>}
    </div>
  );
}
```

这里：

```text
props.title 来自父组件
open 是 Child 自己的状态
```

# 17. useState 和 Zustand 的区别

这个很重要。

| 类型 | 用途 |
|---|---|
| `useState` | 组件自己的临时状态 |
| `Zustand store` | 全局项目状态 |

```text

## Zustand 适合

```

# 18. 对应你的项目：useAppModals

你项目里：

```tsx
const modals = useAppModals();
```

这个 hook 大概率内部用了很多 useState。

比如可能类似：

```tsx
export function useAppModals() {
  const [aiOpen, setAiOpen] = useState(false);
  const [libraryOpen, setLibraryOpen] = useState(false);
  const [cropEditorImageId, setCropEditorImageId] = useState<string | null>(null);
  const [contextMenu, setContextMenu] = useState({
    visible: false,
    x: 0,
    y: 0,
    targetId: null,
  });

  return {
    aiOpen,
    setAiOpen,
    libraryOpen,
    setLibraryOpen,
    cropEditorImageId,
    setCropEditorImageId,
    contextMenu,
    setContextMenu,
  };
}
```

所以：

```tsx
modals.setAiOpen(true)
```

就是打开 AI 弹窗。

```tsx
modals.setCropEditorImageId(id)
```

就是设置当前要裁剪哪张图片，弹窗就会打开。

# 19. 为什么 cropEditorImageId 可以控制弹窗？

假设：

```tsx
const [cropEditorImageId, setCropEditorImageId] = useState<string | null>(null);
```

然后渲染：

```tsx
{cropEditorImageId && (
  <CropEditorModal imageId={cropEditorImageId} />
)}
```

意思：

```text
如果 cropEditorImageId 有值，就显示裁剪弹窗。
如果是 null，就不显示。
```

所以：

```tsx
setCropEditorImageId("img-001")
```

会打开弹窗。

```tsx
setCropEditorImageId(null)
```

会关闭弹窗。

---

# 20. 这和「原生里按 id 决定是否挂载弹窗」是一类事

原生思路：只有 `cropEditorImageId` 有值时才创建/显示弹窗。

React：

```tsx
{cropEditorImageId && (
  <CropEditorModal imageId={cropEditorImageId} />
)}
```

# 23. 用 useState 写一个右键菜单小例子

```tsx
import { useState } from "react";

export function ContextMenuDemo() {
  const [menu, setMenu] = useState({
    visible: false,
    x: 0,
    y: 0,
  });

  return (
    <div
      style={{ height: 300, border: "1px solid #ccc" }}
      onContextMenu={(e) => {
        e.preventDefault();

        setMenu({
          visible: true,
          x: e.clientX,
          y: e.clientY,
        });
      }}
      onMouseDown={() => {
        setMenu((m) => ({ ...m, visible: false }));
      }}
    >
      在这里右键

      {menu.visible && (
        <div
          style={{
            position: "fixed",
            left: menu.x,
            top: menu.y,
            background: "white",
            border: "1px solid #ccc",
            padding: 8,
          }}
        >
          我是右键菜单
        </div>
      )}
    </div>
  );
}
```

这就是你项目右键菜单的最小版。

# 25. useState 常见错误

---

## 错误 1：直接修改对象

不要：

```tsx
menu.visible = false;
setMenu(menu);
```

推荐：

```tsx
setMenu((m) => ({
  ...m,
  visible: false,
}));
```

---

## 错误 2：直接修改数组

不要：

```tsx
items.push("新项");
setItems(items);
```

推荐：

```tsx
setItems([...items, "新项"]);
```

---

## 错误 3：以为 setState 后马上同步变了

```tsx
setCount(count + 1);
console.log(count);
```

这里的 `console.log(count)` 可能还是旧值。

你先简单记住：

```text
setState 会安排一次更新，不要依赖它立刻同步改变。
```
