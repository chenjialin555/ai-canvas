# 第二课：Props 父子组件传值

> 系列目录见 [README.md](./README.md)。

这一课我们只学一个东西：

```text
props
```

你项目里大量代码看不懂，主要就是因为 props。

比如你项目里的：

```tsx
<AppShell
  topBar={<TopBar />}
  leftSidebar={<LeftSidebar />}
  canvasArea={<CanvasArea />}
  rightSidebar={<RightSidebar />}
  overlays={<EditorModals />}
/>
```

还有：

```tsx
<CanvasArea
  stageRef={stageRef}
  onOpenCropEditor={(id) => modals.setCropEditorImageId(id)}
/>
```

这些都是 props。

---

# 1. props 是什么？

一句话：

```text
props 就是父组件传给子组件的数据或函数。
```

用**原生 HTML** 来类比：父页面里写属性，子脚本读属性。

```html
<div id="card" data-name="小明" data-age="18"></div>
```

```js
const el = document.getElementById("card");
const name = el.dataset.name;
const age = Number(el.dataset.age);
```

React 里把「从父到子的输入」写进标签属性，更类型化、更方便复用：

```tsx
<UserCard name="小明" age={18} />
```

子组件收到：

```text
name = 小明
age = 18
```

---

# 2. 为什么需要 props？

因为组件要复用。

比如你想显示很多用户卡片：

```text
小明 18岁
小红 20岁
小刚 22岁
```

你不想写三个不同组件：

```tsx
<XiaoMingCard />
<XiaoHongCard />
<XiaoGangCard />
```

而是写一个通用组件：

```tsx
<UserCard name="小明" age={18} />
<UserCard name="小红" age={20} />
<UserCard name="小刚" age={22} />
```

这就是 props 的作用。

# 3. 最简单的 props 例子

```tsx
function UserCard(props: { name: string; age: number }) {
  return (
    <div>
      <h2>{props.name}</h2>
      <p>年龄：{props.age}</p>
    </div>
  );
}

export function App() {
  return (
    <div>
      <UserCard name="小明" age={18} />
      <UserCard name="小红" age={20} />
    </div>
  );
}
```

---

## 人话解释

```tsx
<UserCard name="小明" age={18} />
```

父组件传给子组件：

```text
name 是 小明
age 是 18
```

子组件里用：

```tsx
props.name
props.age
```

显示出来。

# 4. 和「普通 JS 函数参数」对照

原生思路里，你可能会写一个 **工厂函数**，根据入参生成 DOM：

```js
function renderUserCard(container, { name, age }) {
  container.innerHTML = `
    <div>
      <h2>${name}</h2>
      <p>年龄：${age}</p>
    </div>
  `;
}
```

React 子组件：

```tsx
function UserCard(props: { name: string; age: number }) {
  return (
    <div>
      <h2>{props.name}</h2>
      <p>年龄：{props.age}</p>
    </div>
  );
}
```

对照（帮助记忆）：

| 原生 / 直觉 | React |
|---|---|
| 函数第二个参数 `options` | 函数第一个参数 `props` |
| 模板字符串 `${name}` | JSX 里 `{props.name}` |
| 数字不要加引号 | 数字写 `age={18}`，不要写成 `age="18"`（那是字符串） |

# 5. props 可以传字符串、数字、布尔值

```tsx
function Product(props: {
  title: string;
  price: number;
  available: boolean;
}) {
  return (
    <div>
      <h2>{props.title}</h2>
      <p>价格：{props.price}</p>
      <p>{props.available ? "有货" : "无货"}</p>
    </div>
  );
}

export function App() {
  return (
    <Product
      title="苹果"
      price={5}
      available={true}
    />
  );
}
```

注意：

```tsx
title="苹果"
```

字符串可以直接写。

数字和布尔值要用 `{}`：

```tsx
price={5}
available={true}
```

# 6. props 可以传函数

这是你项目里最常见的。

比如：

```tsx
function Button(props: { onClick: () => void }) {
  return (
    <button onClick={props.onClick}>
      点击
    </button>
  );
}

export function App() {
  return (
    <Button
      onClick={() => alert("父组件传来的函数")}
    />
  );
}
```

---

## 人话解释

父组件说：

```text
Button，你被点击时，帮我执行这个函数。
```

子组件：

```tsx
<button onClick={props.onClick}>
```

意思是：

```text
按钮被点击时，执行父组件传来的 props.onClick。
```

---

# 7. 子组件怎么「通知」父组件？（回调函数）

原生 JS 里常见写法：**父组件传一个函数进来，子组件在事件里调用它**。

```js
function setupChildButton(onOpen) {
  document.querySelector("#childBtn").addEventListener("click", () => {
    onOpen(); // 通知外面
  });
}

setupChildButton(() => {
  console.log("父级收到了：要打开弹窗");
});
```

React 里通常 **不传事件名**，而是直接传函数 props：

父组件：

```tsx
<Child onOpen={openModal} />
```

子组件：

```tsx
function Child(props: { onOpen: () => void }) {
  return (
    <button onClick={props.onOpen}>
      打开
    </button>
  );
}
```

所以：

```text
「子通知父」在 React 里 ≈ 父传下来的回调 props，子负责在合适时机调用。
```

# 12. props 传值方向

props 是从父传子。

```text
父组件
  ↓ props
子组件
```

比如：

```text
App
  ↓ topBar / canvasArea / overlays
AppShell
```

还有：

```text
App
  ↓ onOpenCropEditor
CanvasArea
```

# 13. 子组件怎么通知父组件？

React 里通常通过函数 props。

```text
父组件传一个函数给子组件
子组件在合适的时候调用这个函数
```

例如：

```tsx
function Child(props: { onOpen: () => void }) {
  return <button onClick={props.onOpen}>打开</button>;
}

function Parent() {
  return (
    <Child
      onOpen={() => console.log("子组件通知我了")}
    />
  );
}
```

流程：

```text
父组件 Parent
↓ 把 onOpen 函数传给 Child
Child 用户点击按钮
↓ 调 props.onOpen()
Parent 的函数执行
```

这和上一节的 **`onOpen` 回调**是同一模式：只是有的函数会 **带上参数**（例如 `id`）。

# 14. props 不应该被子组件直接修改

React 里 props 是只读的。

比如：

```tsx
function UserCard(props: { name: string }) {
  props.name = "小红"; // 不要这样
  return <div>{props.name}</div>;
}
```

正确做法是：

```text
如果子组件想改变父组件的数据，
就调用父组件传来的函数。
```

例如：

```tsx
function Child(props: { onChangeName: (name: string) => void }) {
  return (
    <button onClick={() => props.onChangeName("小红")}>
      改名
    </button>
  );
}
```

---

# 15. props 和 state 区别

| 名称 | 谁拥有 | 谁能改 |
|---|---|---|
| props | 父组件传来 | 子组件不能直接改 |
| state | 组件自己拥有 | 组件自己用 setState 改 |
| store | 全局拥有 | 通过 store action 改 |
