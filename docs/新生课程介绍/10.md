这一课解决：

```text

「导出 JSON」导出的到底是什么？
```

> 系列目录见 [README.md](./README.md)。上一课：[9.md](./9.md)。下一课：[11.md](./11.md)。

# 1. 工程数据的根类型：`ProjectJSON`

架构文档里的图：**整个工程**是一个 JSON 对象（概念上），包含例如：

```text
version        — 工程格式版本（迁移用）
savedAt        — 保存时间戳（若有）
pages[]        — 多页；每页有 elements、aiNodes、edges
activePageId   — 当前激活哪一页
```

具体字段以 **`src/editor/types.ts`**（及 `migratePage` 相关）为准。  
你不需要背每一个键：**导入/导出 UI 调用的方法名**会提示你读哪里。

---

# 2. 自动保存：写进浏览器 `localStorage`

第六课提过 **`STORAGE_KEY`**（如 `AI_CANVAS_PRO_PROJECT_V2`）。机制要点：

```text
store 每次 setState 后 → 防抖（约 600ms）→ 尝试 JSON.stringify 后 setItem
```

人话：

```text
你改画布 → 短时间内很多次合并成一次写入 → 减轻卡顿。
```

**注意**：`localStorage` 有大小配额。项目里会对超长 `data:` 图片做瘦身/占位策略，避免 **QuotaExceeded** 把整个交互卡死（见 `persistence/localStoragePersist.ts`）。  
所以：**特别大的图**有时刷新后会变成占位，需要重新导入或重新跑图——这是存储介质限制，不是 React「丢了」。

---

# 3. 选择状态通常不落盘

架构文档说明：`selectedIds` / `selectedWorkflowNodeIds` **默认不写入**持久化包（或加载后清空）。

人话：

```text
你上次选中谁，一般不会帮你记住；
但每一页上的图、节点、连线会记住。
```

---

# 4. 导入 / 导出 /「立即保存」

Store 的 **`projectSlice`** 一类方法里，通常有：

```text
exportProjectJSON / loadProjectJSON — 整块工程的序列化与反序列化
saveLocal — 立即写 localStorage（未必走防抖与瘦身的那条路径，以代码为准）
saveRemote / loadRemote — 若你接了远端存储契约
```

**`loadProjectJSON`** 时往往会走 **`migratePage`**：旧版本工程字段升级成当前 `Page` 形状（例如老的 `workflow` 迁到 `aiNodes`/`edges`）。

---

# 5. 撤销重做与「工程快照」的关系

历史栈里存的 **`Snapshot`** 不仅包含 `pages`，通常还包含 **`activePageId`、`selectedIds`、视口 `zoom`/`pan`** 等（以 `store/types.ts` 为准）。

人话：

```text
Ctrl+Z 恢复的是「当时整包编辑状态」的快照，而不只是某一个元素的 x。
```

工作流节点在 **运行成功/失败收尾** 或 **纯前端同步完成** 时会 `commitHistory`；运行中的 `running` 常 **`history: false`**，避免每一步都占撤销栈。

---

# 6. 和第六课的 Immer 放一起看

`pages` 是很深的树。**`produce`** 让你在 slice 里像改草稿一样写，最后仍得到不可变更新。

读 `elementSlice` / `workflowSlice` 时，把：

```text
set(produce((state) => { ... }))
```

读成：**「改的是 draft，落盘的是新 state」**。

# 9. 本课总结

```text
ProjectJSON = 可搬走的整包工程
localStorage = 浏览器里自动蹲一份（有配额）
migratePage = 老文件升级成当前结构
commitHistory = 撤销栈里的快照，不只有 elements
```
