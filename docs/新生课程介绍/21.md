这一课读 **`src/app/RightSidebar.tsx`**：右侧栏 **不是画布**，全是 **普通 DOM**；负责 **改选中元素属性** 与 **AI 相关侧栏**。

# 1. Tab 从哪来？

`App.tsx` 把 **`useAppModals` 的 `rightTab` / `setRightTab`** 传进来（第十二课）：

```tsx
<RightSidebar
  tab={modals.rightTab}
  onTabChange={modals.setRightTab}
  aiChatAttachmentIds={...}
  onRemoveAiChatAttachment={...}
/>
```

三个枚举值（类型在 **`useAppModals.ts`**）：**`properties` | `aiGenerate` | `aiChat`**。

---

# 2. 三个 Tab 各渲染什么？

| `tab` | 内容（大意） |
|--------|----------------|
| **`properties`** | 未选中 → 提示「请选择元素」；选中 → 名称、类型、图层按钮、几何与样式表单（随 `selected.type` 变） |
| **`aiGenerate`** | **`AiChatPanel`**（带附件 id 列表；与单步弹窗 `AiGenerateModal` 不同壳，同属 AI 能力区） |
| **`aiChat`** | 当前占位「暂无内容」 |

---

# 3. 属性面板怎么改 store？

文件里常见模式：

```tsx
const store = useEditorStore();
const pg = store.getActivePage();
const selected = pg.elements.find((el) => el.id === store.selectedIds[0]);
```

修改时走 **`executeElementCommand({ type: "updateElement", ... })`**（第十三课），而不是在组件里直接改 `selected` 对象 —— 因为 **props/state 来自 store，必须经官方 action 写回**。

另有 **`patch(data, history)`** 包装：`history: false` 用于 **连续拖拽滑块**，在 **`blur` / `mouseup`** 再 **`commitInputGesture()` → `commitHistory()`**，避免撤销一步退十格（注释里写得很清楚）。

---

# 4. 订阅整 store 的取舍

本文件开头使用 **`useEditorStore()`** 无 selector，等价于 **订阅整棵 store**：任何字段变化都可能触发侧栏重渲染。

在 **侧栏本身就要读很多字段** 的写法里有时会出现；若你优化性能，再学 **细粒度 selector**（第六课）也不迟。新手先 **读懂数据从哪来、怎么写回** 即可。

---

# 5. 与 `FloatingToolbar` 的分工

| 区域 | 交互形态 |
|------|----------|
| **浮动条** | 快捷图标，Portal 贴屏幕（第十六课） |
| **右侧栏** | 表单项多、适合 **`input` / `select` 等原生控件** |

两者都可能调 **`executeElementCommand`** 或 **`useEditorStore` 方法**；**不重复造两套业务**，只是入口不同。
