# 第十四课：画布区目录 — 从 `CanvasArea` 到 `StageCanvas` 再到 `canvas/`

这一课只做 **地图**：你打开仓库时，**「画布相关代码」分散在哪些文件夹、谁包谁**。  
不要求你读完每一个 hook，只要知道 **迷路时进哪扇门**。

> 系列目录见 [README.md](./README.md)。上一课：[13.md](./13.md)。下一课：[15.md](./15.md)。

# 1. 最外层：`CanvasArea`（`src/app/CanvasArea.tsx`）

它基本是 **一块 `<section className="canvas-area">`**，里面按顺序摆：

```text
PageTabs        — 多页标签
StageCanvas     — 主 Konva 舞台（核心）
MiniMap         — 小地图
FloatingToolbar — 选中元素后的浮动快捷条（DOM Portal，叠在画布上）
```

**特点**：几乎 **不** 自己订阅 `useEditorStore`；主要把 **App 传来的 props** 继续传给 `StageCanvas` / `FloatingToolbar`（第二课的总线思想）。

人话：

```text
CanvasArea = 「画布这一列 UI」的拼装车间，不是业务大脑。
```

---

# 2. 核心：`StageCanvas`（`src/components/StageCanvas.tsx`）

这里出现 **`<Stage>`（react-konva）**，并在其内部挂三层子结构（名称以源码为准）：

```text
CanvasBackgroundLayer   — 背景层（网格等）
CanvasPageContent       — 当前页：普通元素 + 图片端口叠层 + AI 节点
CanvasInteractionLayer  — 交互层：指针、框选、Transformer 等
```

另外：**拖文件进画布**、**右键菜单**、**空格拖动画布**、**滚轮缩放** 等入口也围绕 `Stage` / 外层 `div` 配置。

**`stageRef`**：仍可在 `App` 里创建，经 `CanvasArea` 传到 `StageCanvas`，用于导出整页等（第四、七课）。

# 3. `src/canvas/` 里有什么？（按子目录记）

| 子目录 / 文件 | 大致职责 |
|----------------|----------|
| **`components/`** | `CanvasBackgroundLayer`、`CanvasPageContent`、`CanvasInteractionLayer` — 被 `StageCanvas` 直接引用的大块 UI |
| **`layers/`** | 更细的 Layer 组合（若架构演进，可能继续拆） |
| **`elements/`** | 每种画布元素怎么画：`ElementNode` 分发到 `ImageElementNode`、`TextElementNode` 等（第十五课） |
| **`interaction/`** | 框选、对齐线、拖线中的临时连线等「画在画布上的交互图形」 |
| **`hooks/`** | 滚轮视口、键盘快捷键、拖入图片、右键菜单 payload、Transformer 逻辑等 **有状态的交互** |
| **`utils/`** | 坐标换算、与 Stage 尺寸相关的纯函数等 |

**记忆口诀**：**`components` 拼大块，`elements` 画元素，`interaction` 画辅助线，`hooks` 管手势，`utils` 算数。**

---

# 4. 和 `src/components/` 的分工（防混）

| 目录 | 典型内容 |
|------|----------|
| `src/components/` | `StageCanvas`、`FloatingToolbar`、`WorkflowNodeView` 等 **跨画布壳子** 或 **与 Stage 强绑定的大组件** |
| `src/canvas/` | **画布坐标系里** 的层、元素节点、画布专用 hook |

边界不必背死：**以 import 为准**；搜 `from "../canvas/` 就能找到谁在用它。

---

# 5. 数据从哪来？

画布上能动的线、图、字，**数据仍在 Zustand**（`pages` → 当前 `activePageId` 的 `elements` / `aiNodes` / `edges`）。  
`canvas/` 里的组件通过 **`useEditorStore(selector)`** 读自己需要的那一小片（第六课）。
