# 第四课：useEffect 和 useRef

这一课我们学两个东西：

```text
1. useEffect：组件显示后做一些事情
2. useRef：保存一个“引用”，不会因为改变而重新渲染
```

你项目里经常看到：

```tsx
useEffect(() => {
  ...
}, []);
```

还有：

```tsx
const stageRef = useRef<Konva.Stage | null>(null);
```

所以这一课非常重要。

---

# 1. 先说结论

## useEffect 是什么？

一句话：

```text
useEffect 用来处理“组件渲染后要做的事情”。
```

比如：

```text
1. 组件加载后请求数据
2. 监听键盘事件
3. 监听窗口 resize
4. 设置定时器
5. 组件销毁时清理事件
```

---

## useRef 是什么？

一句话：

```text
useRef 用来保存一个不会触发页面重新渲染的值。
```

它常用于：

```text
1. 保存 DOM 元素
2. 保存 Konva Stage 实例
3. 保存定时器 id
4. 保存上一次的值
5. 保存拖拽中的临时数据
```

---

# 2. Vue 对比

你之前学 Vue，可以这样对比：

| Vue | React |
|---|---|
| `onMounted()` | `useEffect(() => {}, [])` |
| `onUnmounted()` | `useEffect` 里的 return 清理函数 |
| `watch()` | `useEffect(() => {}, [依赖])` |
| `ref(null)` 获取 DOM | `useRef(null)` |
| `ref()` 保存普通值 | `useState` 或 `useRef`，看是否要触发页面更新 |

---

# 3. 先学 useEffect

最简单写法：

```tsx
import { useEffect } from "react";

export function Demo() {
  useEffect(() => {
    console.log("组件显示到页面上了");
  }, []);

  return <div>Demo</div>;
}
```

---

## 这是什么意思？

```tsx
useEffect(() => {
  console.log("组件显示到页面上了");
}, []);
```

人话：

```text
当这个组件第一次显示到页面上之后，执行里面的代码。
```

它很像 Vue：

```ts
onMounted(() => {
  console.log("组件显示到页面上了")
})
```

---

# 4. useEffect 的基本格式

```tsx
useEffect(() => {
  // 要执行的事情
}, [依赖]);
```

重点是第二个参数：

```tsx
[依赖]
```

它决定什么时候执行。

---

# 5. 三种常见 useEffect

---

## 5.1 只执行一次

```tsx
useEffect(() => {
  console.log("只在组件第一次出现时执行");
}, []);
```

第二个参数是空数组：

```tsx
[]
```

意思是：

```text
只执行一次。
```

类似 Vue：

```ts
onMounted(() => {})
```

---

## 5.2 某个值变化时执行

```tsx
useEffect(() => {
  console.log("count 变了：", count);
}, [count]);
```

意思：

```text
组件第一次显示时会执行一次。
之后每次 count 改变，也会执行。
```

类似 Vue：

```ts
watch(count, () => {
  console.log("count 变了")
})
```

---

## 5.3 每次渲染都执行

```tsx
useEffect(() => {
  console.log("组件每次渲染都会执行");
});
```

没有第二个参数。

这个不常用，因为容易造成性能问题。

你先记住：

```text
大多数情况下，不要省略第二个参数。
```

---

# 6. 用 useEffect 监听键盘事件

这是你项目里很常见的场景，比如：

```text
Ctrl+Z 撤销
Ctrl+C 复制
Ctrl+V 粘贴
空格拖动画布
Delete 删除
```

简单例子：

```tsx
import { useEffect } from "react";

export function KeyboardDemo() {
  useEffect(() => {
    function handleKeyDown(e: KeyboardEvent) {
      console.log("按下了：", e.key);
    }

    window.addEventListener("keydown", handleKeyDown);

    return () => {
      window.removeEventListener("keydown", handleKeyDown);
    };
  }, []);

  return <div>请按键盘</div>;
}
```

---

## 重点看这几行

```tsx
window.addEventListener("keydown", handleKeyDown);
```

意思：

```text
监听键盘按下事件。
```

---

```tsx
return () => {
  window.removeEventListener("keydown", handleKeyDown);
};
```

意思：

```text
组件销毁时，移除监听。
```

---

# 7. 为什么要清理事件？

如果你不清理：

```tsx
window.addEventListener("keydown", handleKeyDown);
```

组件卸载后，这个监听可能还在。

后果：

```text
1. 内存泄漏
2. 一个按键触发多次
3. 页面越来越卡
```

所以有监听，就要清理。

---

# 8. useEffect 的清理函数

格式：

```tsx
useEffect(() => {
  // 创建副作用

  return () => {
    // 清理副作用
  };
}, []);
```

常见需要清理的东西：

```text
1. window.addEventListener
2. setInterval
3. setTimeout
4. websocket
5. 外部库实例
```

# 9. 定时器例子

```tsx
import { useEffect, useState } from "react";

export function TimerDemo() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    const timer = window.setInterval(() => {
      setCount((c) => c + 1);
    }, 1000);

    return () => {
      window.clearInterval(timer);
    };
  }, []);

  return <div>秒数：{count}</div>;
}
```

人话：

```text
组件出现后，每秒 count +1。
组件销毁时，清掉定时器。
```

# 10. 再学 useRef

最简单写法：

```tsx
import { useRef } from "react";

export function RefDemo() {
  const inputRef = useRef<HTMLInputElement | null>(null);

  return (
    <div>
      <input ref={inputRef} />
      <button
        onClick={() => {
          inputRef.current?.focus();
        }}
      >
        聚焦输入框
      </button>
    </div>
  );
}
```

---

## 这是什么意思？

```tsx
const inputRef = useRef<HTMLInputElement | null>(null);
```

人话：

```text
创建一个 ref，用来保存 input 元素。
初始值是 null。
```

---

```tsx
<input ref={inputRef} />
```

意思：

```text
把真实的 input DOM 元素放到 inputRef.current 里。
```

---

```tsx
inputRef.current?.focus();
```

意思：

```text
如果 inputRef.current 存在，就调用它的 focus 方法。
```

# 11. useRef 的 current

`useRef` 返回的是一个对象：

```ts
{
  current: 初始值
}
```

所以你经常看到：

```tsx
stageRef.current
inputRef.current
timerRef.current
```

`current` 才是真正保存的东西。

---

# 12. useRef 和 useState 的区别

这是重点。

| 对比 | useState | useRef |
|---|---|---|
| 改变后页面会重新渲染吗 | 会 | 不会 |
| 用途 | 页面要显示的状态 | 保存引用或临时值 |
| 访问方式 | `count` | `ref.current` |
| 修改方式 | `setCount(...)` | `ref.current = ...` |

---

## 例子

### useState

```tsx
const [count, setCount] = useState(0);
setCount(count + 1);
```

会导致页面重新渲染。

---

### useRef

```tsx
const countRef = useRef(0);
countRef.current += 1;
```

不会导致页面重新渲染。

---

# 13. 为什么需要不会触发渲染的值？

比如定时器 id：

```tsx
const timerRef = useRef<number | null>(null);
```

你只是想保存这个 id，方便以后清理。

你不需要它显示到页面上。

所以用 `useRef`，不用 `useState`。

# 16. useRef 保存临时拖拽数据

画布项目里，经常用 `useRef` 保存拖拽中的临时数据。

比如：

```tsx
const draggingRef = useRef(false);
```

拖动开始：

```tsx
draggingRef.current = true;
```

拖动结束：

```tsx
draggingRef.current = false;
```

为什么不用 state？

因为拖拽中变化很频繁。

如果用 state：

```tsx
setDragging(true)
setDragging(false)
```

可能导致组件重新渲染。

而有些临时值只是给事件函数用，不需要显示出来，所以用 ref。

# 17. useEffect + useRef 配合

例如：定时器。

```tsx
import { useEffect, useRef } from "react";

export function TimerRefDemo() {
  const timerRef = useRef<number | null>(null);

  useEffect(() => {
    timerRef.current = window.setTimeout(() => {
      console.log("2 秒后执行");
    }, 2000);

    return () => {
      if (timerRef.current !== null) {
        window.clearTimeout(timerRef.current);
      }
    };
  }, []);

  return <div>TimerRefDemo</div>;
}
```

人话：

```text
timerRef 保存定时器 id。
组件销毁时根据这个 id 清理定时器。
```

# 18. useEffect 依赖数组怎么理解？

这是很多人容易混的地方。

```tsx
useEffect(() => {
  console.log(count);
}, [count]);
```

这里的 `[count]` 意思是：

```text
这个 effect 依赖 count。
count 变化时重新执行。
```

---

## 例子

```tsx
import { useEffect, useState } from "react";

export function CountEffectDemo() {
  const [count, setCount] = useState(0);
  const [name, setName] = useState("小明");

  useEffect(() => {
    console.log("count 变化了：", count);
  }, [count]);

  return (
    <div>
      <button onClick={() => setCount(count + 1)}>
        count + 1
      </button>

      <button onClick={() => setName("小红")}>
        改名字
      </button>
    </div>
  );
}
```

这里：

```text
点击 count + 1 → effect 执行
点击改名字 → effect 不执行
```

因为依赖数组里只有 `count`。

# 19. useEffect 常见错误

---

## 错误 1：忘记依赖

```tsx
useEffect(() => {
  console.log(count);
}, []);
```

如果你希望 count 改变时执行，那这里就错了。

应该：

```tsx
useEffect(() => {
  console.log(count);
}, [count]);
```

---

## 错误 2：没有清理事件

```tsx
useEffect(() => {
  window.addEventListener("keydown", handleKeyDown);
}, []);
```

应该：

```tsx
useEffect(() => {
  window.addEventListener("keydown", handleKeyDown);

  return () => {
    window.removeEventListener("keydown", handleKeyDown);
  };
}, []);
```

---

## 错误 3：在 effect 里无限 setState

```tsx
useEffect(() => {
  setCount(count + 1);
}, [count]);
```

这会导致：

```text
count 变 → effect 执行 → setCount → count 又变 → effect 又执行
```

可能无限循环。

# 20. React StrictMode 和 useEffect

你的 `main.tsx` 里有：

```tsx

  <App />
</StrictMode>
```

在开发环境下，React StrictMode 可能会让 effect 执行两次。

所以如果你看到：

```text

```

不一定是 bug。

生产环境一般不会这样。

# 24. useRef 为什么不会导致页面更新？

比如：

```tsx
const valueRef = useRef(0);

<button onClick={() => {
  valueRef.current += 1;
}}>
  增加
</button>
```

点击后：

```text
valueRef.current 变了
但页面不会自动刷新
```

因为 React 不知道你想重新显示。

如果你希望页面显示变化，应该用 `useState`：

```tsx
const [value, setValue] = useState(0);
```

# 25. 什么时候用 useState，什么时候用 useRef？

你可以用这个规则：

## 页面要显示这个值？

用 `useState`。

```tsx
const [count, setCount] = useState(0);
```

因为 count 变了，页面要显示新 count。

---

## 只是内部临时保存，不需要显示？

用 `useRef`。

```tsx
const timerRef = useRef<number | null>(null);
```

比如：

```text
定时器 id
拖拽中标记
Stage 实例
上一帧鼠标位置
```

# 27. 小练习 2：监听键盘

```tsx
import { useEffect, useState } from "react";

export function KeyDemo() {
  const [key, setKey] = useState("");

  useEffect(() => {
    function handleKeyDown(e: KeyboardEvent) {
      setKey(e.key);
    }

    window.addEventListener("keydown", handleKeyDown);

    return () => {
      window.removeEventListener("keydown", handleKeyDown);
    };
  }, []);

  return <div>你按下了：{key}</div>;
}
```

你要理解：

```text
useEffect 负责注册键盘监听。
按键后 setKey。
key 更新后页面显示新按键。
```

# 29. 小练习 4：useRef 保存不渲染的值

```tsx
import { useRef, useState } from "react";

export function RefCountDemo() {
  const refCount = useRef(0);
  const [stateCount, setStateCount] = useState(0);

  return (
    <div>
      <p>stateCount：{stateCount}</p>
      <p>refCount 不会自动显示变化：{refCount.current}</p>

      <button
        onClick={() => {
          refCount.current += 1;
          console.log("refCount:", refCount.current);
        }}
      >
        增加 refCount
      </button>

      <button onClick={() => setStateCount(stateCount + 1)}>
        增加 stateCount
      </button>
    </div>
  );
}
```

你会发现：

```text
点 refCount 按钮，console 变了，但页面可能不刷新。
点 stateCount 按钮，页面刷新。
```

这说明：

```text
useRef 改变不会触发重新渲染。
useState 改变会触发重新渲染。
```

# 32. 本课总结

今天你只要记住：

```text
useEffect：组件渲染后做事，比如监听事件、请求数据、设置定时器。
useRef：保存引用或临时值，改变它不会自动刷新页面。
```

最常见格式：

```tsx
useEffect(() => {
  // 做事

  return () => {
    // 清理
  };
}, []);
```

和：

```tsx
const ref = useRef(null);
ref.current
```
