# 第五课：TypeScript 最小够用知识

这一课我们只学你看这个项目需要的 TypeScript 基础。

你不需要先学完 TypeScript 全部内容。  
你现在只需要能看懂这些：

```tsx
type AppShellProps = {
  onRootMouseDown: () => void;
  topBar: ReactNode;
};

const stageRef = useRef<Konva.Stage | null>(null);

function updateElement(id: string, patch: Partial<CanvasElement>) {}
```

这一课目标：

```text
看到类型不害怕，能大概读懂它在说什么。
```

---

# 1. TypeScript 是什么？

你之前学的是 JavaScript。

JavaScript：

```js
const name = "小明";
const age = 18;
```

TypeScript 是 JavaScript 的增强版，它给变量加了“类型说明”。

TypeScript：

```ts
const name: string = "小明";
const age: number = 18;
```

人话：

```text
name 是字符串
age 是数字
```

---

# 2. 为什么要用 TypeScript？

因为项目大了以后，数据结构很复杂。

比如一个画布元素：

```ts
{
  id: "abc",
  type: "image",
  x: 100,
  y: 200,
  width: 300,
  height: 200,
  src: "xxx.png"
}
```

如果没有类型，你很容易写错：

```ts
element.wdith // 拼错 width
element.srcUrl // 根本没有这个字段
```

TypeScript 可以提前报错。

| `number` | 数字 |

# 4. 数组类型

```ts
const names: string[] = ["小明", "小红"];
const ages: number[] = [18, 20, 22];
```

意思：

```text
names 是字符串数组
ages 是数字数组
```

另一种写法：

```ts
const names: Array<string> = ["小明", "小红"];
```

你先记第一种：

```ts
string[]
number[]
```

比如一个用户：

```ts

  id: string;
  name: string;
  age: number;

```

意思：

```text
User 这种对象必须有：
- id，字符串
- name，字符串
- age，数字
```

使用：

```ts
const user: User = {
  id: "u1",

```

```ts
type User = {
  id: string;
  name: string;

```

`type` 可以理解为：


比如：


# 8. 可选字段 `?`

```ts
type User = {
  id: string;
  name: string;
  avatar?: string;
};
```

这里：

```ts
avatar?: string
```

意思是：

```text
avatar 可以有，也可以没有。
如果有，必须是字符串。
```

比如这样可以：

```ts
const user1: User = {
  id: "1",
  name: "小明",
};
```

也可以：

```ts
const user2: User = {
  id: "2",
  name: "小红",
  avatar: "avatar.png",
};
```

# 9. 联合类型 `|`

```ts
type Tool = "select" | "pan" | "text";
```

意思是：

```text
Tool 只能是：
- select
- pan
- text
这三个值之一。
```

使用：

```ts
const tool: Tool = "select"; // 对
const tool2: Tool = "abc";   // 错
```

你的项目里会有很多这种类型：

```ts
type ToolType = "select" | "pan" | "text" | "rect";
```

# 10. null 类型

```ts
const selectedId: string | null = null;
```

意思是：

```text
selectedId 可以是字符串，也可以是 null。
```

例如：

```ts
let selectedId: string | null = null;

selectedId = "abc"; // 可以
selectedId = null;  // 也可以
```

你的项目里很多状态会这样：

```ts
cropEditorImageId: string | null
```

意思：

```text
如果是 string，表示正在裁剪某张图。
如果是 null，表示没有打开裁剪弹窗。
```

# 11. 函数参数类型

JavaScript：

```js
function add(a, b) {
  return a + b;
}
```

TypeScript：

```ts
function add(a: number, b: number): number {
  return a + b;
}
```

意思：

```text
a 是数字
b 是数字
返回值也是数字
```


`as` 可以先理解为：


```text

```

# 18. Partial 是什么？

你可能看到：

```ts
Partial<CanvasElement>
```

`Partial<T>` 意思是：

```text
把 T 里的所有字段都变成可选。
```

比如：

```ts
type User = {
  id: string;
  name: string;
  age: number;
};
```

那么：

```ts
Partial<User>
```

等价于：

```ts
{
  id?: string;
  name?: string;
  age?: number;
}
```

---

## 为什么 updateElement 用 Partial？

比如一个元素有很多字段：

```ts
{
  id,
  type,
  x,
  y,
  width,
  height,
  rotation,
  opacity,
  ...
}
```

你更新时不想每次传完整对象。

你只想改一个字段：

```ts
updateElement(id, { x: 100 })
```

所以第二个参数是：

```ts
Partial<CanvasElement>
```

意思是：

```text
你可以只传 CanvasElement 的一部分字段。
```

# 19. Pick 是什么？

你可能看到：

```ts
Pick<ImageElement, "cropOffsetX" | "cropOffsetY">
```

`Pick<T, K>` 意思是：

```text
从 T 里挑几个字段组成新类型。
```

比如：

```ts
type User = {
  id: string;
  name: string;
  age: number;
};

type UserName = Pick<User, "id" | "name">;
```

等价于：

```ts
type UserName = {
  id: string;
  name: string;
};
```

---

# 20. Omit 是什么？

```ts
Omit<User, "age">
```

意思：

```text
从 User 里去掉 age 字段。
```

比如：

```ts
type UserWithoutAge = {
  id: string;
  name: string;
};
```

你现在只需要知道：

```text
Partial = 所有字段变可选
Pick = 挑字段
Omit = 去掉字段
```

# 28. 可选链 `?.`

```ts
user?.name
```

意思：

```text
如果 user 存在，取 user.name。
如果 user 不存在，结果是 undefined。
```

等价于：

```ts
user ? user.name : undefined
```

你的项目里经常会用：

```ts
selected?.type === "image"
```

意思：

```text
如果 selected 存在，再判断它的 type。
如果 selected 不存在，不报错。
```

# 29. 非空断言 `!`

你的 `main.tsx`：

```ts
document.getElementById("root")!
```

这里的 `!` 是 TypeScript 非空断言。

意思：

```text
我确定这里不是 null。
```

因为 TypeScript 会担心：

```ts
document.getElementById("root")
```

可能找不到元素，返回 null。

但你知道 index.html 里一定有：

```html
<div id="root"></div>
```

所以写：

```ts
!
```

告诉 TypeScript 别担心。

# 32. 这节课你要记住的 10 个东西

```text
1. string：字符串
2. number：数字
3. boolean：布尔
4. type：给数据结构起名字
5. ?: 可选字段
6. |：联合类型，可以是这个或那个
7. () => void：无参数无返回函数

10. import type：只导入类型
```
