# 第八课：开发时前后端怎么连上（Vite 代理与 `/api`）

> 系列目录见 [README.md](./README.md)。上一课：[7.md](./7.md)（Konva）。下一课：[9.md](./9.md)（画布工作流节点）。

这一课面向：**主要写前端、但项目是前后端分离** 的你。  
我们只讲 **本地开发时** 浏览器里发生了什么，不讲部署运维。

---

# 1. 你的项目运行时有几「层」？

根据 `README.md` 与架构文档，本地典型是：

```text
浏览器打开 http://127.0.0.1:5173
  → 这是 Vite 开发服务器（前端）
  → 前端代码里请求 /api/...
  → Vite 把 /api 代理到后端（端口见下）
  → FastAPI 处理后再把 JSON 返回给浏览器
```

**后端端口怎么对齐？** 看仓库根目录 `vite.config.ts`：`/api` 代理到 `http://localhost:${apiPort}`，其中 `apiPort = process.env.API_PORT`（来自 `.env`）**或** 默认 **`13555`**（与注释里「与 package.json 里 uvicorn 端口一致」对齐）。你改了 `.env` 里的 `API_PORT` 时，**前后端要一起改**，否则代理会指错端口。

人话：

```text
你眼睛看到的是「一个网站」，
背后其实是「两个进程在对话」。
```

---

# 2. 为什么需要代理？（跨域直觉）

浏览器的**同源策略**大致说：

```text
页面在 http://127.0.0.1:5173
如果 JS 直接去请求 http://127.0.0.1:13555
有可能被浏览器拦（CORS）。
```

**开发代理**的做法是：

```text
让浏览器始终只访问 5173，
/api/* 由 Vite 在服务器侧转发到 13555，
对浏览器来说还是「同源」。
```

具体规则写在 `vite.config.ts` 里（你改端口时要对照 `.env` 与脚本）。

---

# 3. 前端代码里怎么发请求？

项目里 AI 相关逻辑集中在 `src/ai/` 等目录，路径通常写成 **`/api/...`**（相对当前站点根）。

人话：

```text
不要在前端写死 http://127.0.0.1:13555/...
而是写 /api/...，交给 Vite 代理。
```

这样 **开发** 与 **以后部署到同一域名** 时都更省事。

---

# 4. 单步生图 vs 工作流节点（你只要知道「有两条路」）

架构文档里概括过：

| 能力 | HTTP 例子（概念） | 说明 |
|------|------------------|------|
| 单步 AI 弹窗生图 | `POST /api/generate-image` 等 | 表单填完 → 请求 → OSS/网关 |
| 画布工作流节点 | `POST /api/workflow/run-node` | 某些节点类型纯前端；其它走后端 executor |

你是前端新手时，**先不用读完所有 Python**。  
只要知道：**按钮点下去 → 某个 `fetch` / 封装函数 → 后端返回 JSON → store 或组件 state 更新界面**。

---

# 5. `concurrently` 是干什么的？

`npm run dev` 往往同时拉起 **Vite + uvicorn**（以你 `package.json` 脚本为准）。

人话：

```text
一条命令起两个服务，省得你开两个终端手动记顺序。
```

---

# 6. 环境变量 `.env` 你要关心什么？

`README` 说复制 `.env.example` → `.env`。  
常见会配：

```text
后端监听端口
模型/网关 Key
OSS 相关（上传图）
```

原则：

```text
不要把含密钥的 .env 提交到 git。
前端能看到的变量要假设「用户也能看到」（Vite 暴露给浏览器的那部分）。
```

细节以 `.env.example` 与 `backend/app/core/settings.py` 为准。

# 7. 练习：在 DevTools 里自己验证代理

你要能按步骤做（口述即可）：

```text
1. 打开网站，按 F12 → Network
2. 触发一次会请求后端的功能（如拉模型列表、生图）
3. 看到请求 URL 是 /api/...，状态码 200 或对应错误
4. 点进去看 Response 是不是 JSON
```
