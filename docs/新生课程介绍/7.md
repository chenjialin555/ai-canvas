# 1. 为什么不用一堆 `<div>` 做画布？

如果每张图、每个矩形、每条线都是一个 DOM 节点：

```text
元素一多，浏览器排版和事件分发会变慢。
```

**Konva** 用 **`<canvas>`**（可理解为一块位图 + 自己的场景图）来画：  
内部维护一棵「图形树」，**不是** React 那棵 DOM 树。

人话：

```text
Figma / 画图软件类的产品，常用 canvas 技术栈。
你的项目选的是 Konva（2D canvas 场景库）。
```

---

# 2. Konva 里最小的世界观

几个词先记住：

| 概念 | 人话 |
|------|------|
| **Stage** | 舞台：对应一块 canvas 视口，接收鼠标键盘 |
| **Layer** | 层：一层一个（或几个）canvas，分层绘制、控制性能 |
| **Node / Shape** | 树上的节点：矩形、图片、文字路径、组… |
| **Group** | 容器：里面再放子节点，像「文件夹」 |

你的主舞台在架构文档里写的是 **大约 3 个 Layer**：背景、内容、交互装饰（对齐线、框选、Transformer 等）。  
具体以 `StageCanvas` 引用的 `canvas/components/*` 为准。

---

# 3. react-konva 是什么？

**Konva** 本身是偏命令式的 API（`new Konva.Stage(...)`）。  
**react-konva** 提供 React 组件包装：

```tsx
import { Stage, Layer } from "react-konva";
```

人话：

```text
你用 JSX 写 <Stage><Layer>...</Layer></Stage>，
react-konva 在底层帮你创建/更新 Konva 对象。
```

---

# 4. 和 React 的关系：谁管「状态」？

**数据（元素列表、坐标、选中）仍然在 Zustand（`useEditorStore`）里。**  
**Konva 负责「把这些数据画出来 + 接住指针事件」。**

典型分工：

```text
Store：真相源（JSON 可序列化的那套）
React：布局壳、侧栏、弹窗、订阅 store
Konva：高性能绘制 + 变换手柄（Transformer）
```

---

# 5. `stageRef` 是干什么的？（第四课复习 + 落地）

你在 `App.tsx` 里见过：

```tsx
const stageRef = useRef<Konva.Stage | null>(null);
```

在 `StageCanvas` 里会把真实 `Konva.Stage` 实例塞进去（通过 `ref` 回调或外部 ref）。

**用途举例**（项目里真实存在的需求）：

```text
导出整张画布为 PNG → 需要拿到 Stage 或 layer 的 toDataURL
把鼠标屏幕坐标换算成「画布世界坐标」→ 需要读 Stage 的 scale / position
```

人话：

```text
有些 API 必须对着「真实的 Konva Stage 对象」调用，
所以要有 ref 保存它。
```

---

# 6. 世界坐标 vs 屏幕坐标（必须建立心智模型）

架构文档里的公式（和 `StageCanvas` 里指针换算一致）：

```text
worldX = (pointerX - pan.x) / zoom
worldY = (pointerY - pan.y) / zoom
```

直觉：

```text
pan  把整张画布平移
zoom 把整张画布缩放

Store 里保存的元素 x/y/width/height，是世界坐标。
右键菜单、浮动条用 fixed 定位，多是屏幕坐标。
```

所以你读代码时看到 **既在算 Stage 上的点，又在算 DOM 浮层的位置**，不要慌：  
**它们本来就不是同一个坐标系**，中间要做换算。

---

# 7. Transformer 是什么？

Konva 的 **`Transformer`** 是「选中框 + 缩放点 + 旋转手柄」那套 UI。  
你的项目里它放在**交互层**，避免挡住内容层里真正的图片/形状。

你只要知道：

```text
Transformer 绑在「当前选中的 Konva 节点」上，
拖拽结束后再把新的宽高/位置写回 store（往往走 updateElement）。
```

---

# 8. 事件从 Konva 冒泡到业务的大致路径

简化成：

```text
用户操作 Konva
  → Stage/Layer 上的 onClick / onDragEnd / ...
  → 调 useEditorStore.getState().xxx(...) 或 hook 里已捕获的 action
  → store 更新
  → React 组件因 selector 订阅而重渲染
  → react-konva 根据新 props 更新 Konva 树
```

中间可能有 **「手势合并历史」** 之类的优化，但大方向就是这样。

worldX = (300 - 100) / 2 = 100

# 14. 本课总结

```text
Konva       = canvas 上的 2D 场景图 + 事件

Store       = 存世界坐标与业务数据
stageRef    = 拿到真实 Stage，做导出等 imperative 操作

```
