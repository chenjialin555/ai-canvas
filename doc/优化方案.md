# 任务 1：检查还有没有“老写法”

白话解释：

你现在做了一个“统一操作入口”。

以前可能是：

```text
这个按钮自己改元素
那个菜单自己改元素
右侧属性栏又自己改元素
```

这样容易乱。

现在你已经开始改成：

```text
所有按钮都走同一个操作入口
```

这样撤销、锁定、隐藏都更好控制。

下一步就是检查：

```text
还有没有漏掉的按钮没改？
```

比如：

```text
1. 右键菜单
2. 顶部工具栏
3. 浮动工具栏
4. 右侧属性栏
5. 快捷键
6. 图层面板
```

这件事的目的：

```text
让所有编辑操作都统一，后面不容易出 bug。
```

---

# 任务 2：把“图片裁剪”代码整理干净

白话解释：

你项目里有图片裁剪功能。

比如用户点一张图片，可以裁剪它。

裁剪里面有很多计算：

```text
1. 图片怎么缩放
2. 裁剪框怎么移动
3. 裁剪框不能超出图片
4. 鼠标滚轮缩放怎么算
5. 拖动边框怎么算
```

这些计算如果都写在界面文件里，会很乱。

你现在已经拆出一部分了。  
下一步就是：

```text
把剩下的裁剪计算也都放到专门的文件里。
```

以后你想加：

```text
1:1 裁剪
16:9 裁剪
自由裁剪
圆角裁剪
```

会容易很多。

---

# 任务 3：把“画蒙版”代码整理干净

白话解释：

蒙版就是用户在图片上用画笔涂一块区域，然后让 AI 改这块区域。

比如：

```text
用户涂掉衣服 → AI 换衣服
用户涂掉背景 → AI 换背景
用户涂掉物体 → AI 删除物体
```

蒙版里面也有很多逻辑：

```text
1. 鼠标按下开始画
2. 鼠标移动继续画
3. 鼠标松开结束
4. 笔刷大小
5. 橡皮擦
6. 清空蒙版
7. 反选蒙版
```

现在建议做的是：

```text
把“画笔逻辑”从界面里拆出来。
```

这样后面加橡皮擦、清空、反选会更容易。

# 任务 4：让 AI 工作流可以“运行上游”

这个比较像真正的新功能。

现在工作流可能是这样：

```text
图片输入 → AI 处理 → 放大 → 输出
```

用户现在可能需要一个一个节点点“运行”。

你可以加一个功能：

```text
右键某个节点，选择“运行上游”
```

意思是：

```text
自动把这个节点前面的所有节点按顺序跑完。
```

比如：

```text
A → B → C
```

用户右键 C，点“运行上游”，软件自动跑：

```text
A
B
C
```

这个功能非常实用。

# 任务 5：给项目文件加“版本号”

白话解释：

你现在项目可以保存成 JSON 文件。

但是以后你会不断加字段，比如：

```text
锁定
隐藏
AI 节点结果
运行日志
素材资源
```

旧文件可能没有这些字段。

如果没有版本机制，以后可能出现：

```text
旧项目打不开
导入报错
字段缺失导致页面白屏
```

所以可以给保存文件加一个版本号：

```json
{
  "version": 2,
  "pages": []
}
```

然后代码里判断：

```text
如果是旧版本，就自动补齐新字段。
```

这叫“项目文件迁移”。

你可以理解成：

```text
让老项目文件也能在新版本软件里打开。
```
