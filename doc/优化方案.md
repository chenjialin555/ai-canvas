明白了。你前面的**结构性重构阶段已经完成**，现在目标不是“再拆文件”，而是：

# 画布性能优化专项

重点不是架构，而是：

```text
减少不必要重渲染
减少高频 store 写入
减少 Konva 绘制压力
减少图片和节点带来的性能负担
减少鼠标移动/滚轮/拖拽时的计算量
```

# 一、先给你结论：最优先优化这 8 件事

按照收益从高到低：

```text
1. 禁止组件整包订阅 useEditorStore()
2. 拖拽/缩放/框选过程中不要频繁写 store
3. pan / zoom 用 requestAnimationFrame 合并更新
4. Konva 不需要交互的节点全部 listening={false}
5. 背景网格、辅助线、连线、装饰层做缓存或静态化
6. 图片降采样 / 缩略图 / 避免大图直接参与实时绘制
7. MiniMap 降频更新
8. React.memo / useMemo / useCallback 包住高频组件和 props
```

你可以把它当作“画板丝滑八件套”。

# 三、性能优化第一阶段：先找出“谁在频繁重渲染”

这一步很重要。不要盲改。

---

## 1. 检查是否有整包订阅 store

搜索：

```ts
useEditorStore()
```

尤其是这种：

```ts
const store = useEditorStore();
```

这个很危险。

因为它表示：

```text
只要 store 任何字段变了，这个组件就重新渲染。
```

应该改成：

```ts
const selectedIds = useEditorStore((s) => s.selectedIds);
const setSelectedIds = useEditorStore((s) => s.setSelectedIds);
const updateElement = useEditorStore((s) => s.updateElement);
```

也就是：

```text
用多少，订阅多少。
```

## 重点文件优先查这些

```text
src/app/App.tsx
src/app/CanvasArea.tsx
src/components/StageCanvas.tsx
src/canvas/layers/ContentLayer.tsx

src/canvas/elements/ImageElementNode.tsx
src/components/workflow/WorkflowNodeView.tsx
src/components/MiniMap.tsx
src/components/FloatingToolbar.tsx
```

如果这些文件里有：

```ts
const store = useEditorStore();
```

优先改掉。

## 2. 给关键组件临时加渲染日志

```tsx
console.count("StageCanvas render");
```

# 四、性能优化第二阶段：拆 store 订阅，减少 React 重渲染

这是最值得先做的。

---

## 问题写法

```ts
const store = useEditorStore();
```

或者：

```ts
const {
  pages,
  selectedIds,
  zoom,
  pan,
  updateElement,
  setSelectedIds,
} = useEditorStore();
```

这会导致组件订阅整个 store。

---

## 推荐写法

```ts
const pages = useEditorStore((s) => s.pages);
const activePageId = useEditorStore((s) => s.activePageId);
const selectedIds = useEditorStore((s) => s.selectedIds);
const zoom = useEditorStore((s) => s.zoom);
const pan = useEditorStore((s) => s.pan);

const updateElement = useEditorStore((s) => s.updateElement);
const setSelectedIds = useEditorStore((s) => s.setSelectedIds);
```

---

## 更好：用 selector

比如创建：

```text
src/editor/store/selectors/pageSelectors.ts
```

```ts
export const selectActivePage = (s: Store) =>
  s.pages.find((p) => p.id === s.activePageId);
```

然后：

```ts
const activePage = useEditorStore(selectActivePage);
```

---

## 对复杂对象用 shallow

如果一次取多个字段：

```ts
import { shallow } from "zustand/shallow";

const { zoom, pan } = useEditorStore(
  (s) => ({
    zoom: s.zoom,
    pan: s.pan,
  }),
  shallow,
);
```

这样可以减少无意义更新。

> 注意：Zustand v5 的 shallow 用法可能根据版本略有区别，有些项目用 `useShallow`。

# 五、性能优化第三阶段：高频操作不要频繁写 store

这是画布丝滑的核心。

---

# 1. 拖动元素优化

## 不推荐

```tsx
onDragMove={(e) => {
  updateElement(id, {
    x: e.target.x(),
    y: e.target.y(),
  });
}}
```

因为 `dragmove` 每秒可能触发几十次。

---

## 推荐

```tsx
onDragMove={() => {
  // 不写 store，让 Konva 自己移动
}}

onDragEnd={(e) => {
  updateElement(id, {
    x: e.target.x(),
    y: e.target.y(),
  });
}}
```

也就是：

```text
拖动中：Konva 自己动
拖动结束：一次写回 store
```

---

# 2. 缩放/旋转元素优化

## 不推荐

```tsx
onTransform={(e) => {
  updateElement(...);
}}
```

---

## 推荐

```tsx
onTransform={() => {
  // 不写 store
}}

onTransformEnd={(e) => {
  // 最后一次性写回 width/height/x/y/rotation
}}
```

---

# 3. 框选优化

框选时不要每次 mousemove 都：

```ts
setSelectedIds(...)
```

应该：

```text
mousemove 只更新框选矩形
mouseup 时再计算最终 selectedIds
```

如果需要实时高亮，也要用 requestAnimationFrame 或节流。

---

# 4. 连线拖拽优化

AI 节点连线时：

```text
临时线坐标不要每个 mousemove 都进 store
```

最好放：

```text
组件本地 state/ref
或者 Konva 临时 node
```

结束连线时再写入：

```text
Page.edges
```

---

# 六、性能优化第四阶段：pan / zoom 用 requestAnimationFrame 合并

滚轮和拖动画布是高频事件。

如果每次 wheel 都直接：

```ts
setZoom(newZoom);
setPan(newPan);
```

会卡。

---

## 推荐做一个 raf 调度器

可以做一个 hook：

```text
src/canvas/hooks/useRafViewport.ts
```

伪代码：

```ts
const frameRef = useRef<number | null>(null);
const pendingRef = useRef<{ zoom: number; pan: Point } | null>(null);

const scheduleViewportUpdate = (next: { zoom: number; pan: Point }) => {
  pendingRef.current = next;

  if (frameRef.current !== null) return;

  frameRef.current = requestAnimationFrame(() => {
    const pending = pendingRef.current;
    if (pending) {
      setZoom(pending.zoom);
      setPan(pending.pan);
    }

    pendingRef.current = null;
    frameRef.current = null;
  });
};
```

这样：

```text
一帧最多更新一次 pan/zoom
```

会明显更顺。

---

## 更进一步

如果还卡，可以考虑：

```text
拖动画布中直接改 Konva Stage 的 position
拖动结束再同步 store
```

也就是：

```text
交互过程中走 Konva imperative
交互结束写 Zustand
```

但这个会复杂一点，建议第二轮再做。

---

# 七、性能优化第五阶段：优化 Konva 节点

Konva 的性能和 DOM 不一样。  
它主要卡在：

```text
绘制
事件命中检测
图片处理
滤镜
阴影
缓存
```

---

# 1. 不需要事件的节点全部加 listening={false}

这是最简单有效的优化之一。

---

## 背景网格

```tsx
<Layer listening={false}>
  <BackgroundGrid />
</Layer>
```

---

## 辅助线

```tsx
<Line listening={false} />
```

---

## 文字装饰

```tsx
<Text listening={false} />
```

---

## 连线如果不需要点击

```tsx
<Line listening={false} />
```

如果连线需要点击删除，可以只让“透明加粗命中线”监听事件，视觉线不监听。

```tsx
<Group>
  <Line
    points={points}
    stroke="#888"
    strokeWidth={2}
    listening={false}
  />
  <Line
    points={points}
    stroke="transparent"
    strokeWidth={12}
    onClick={...}
  />
</Group>
```

---

# 2. 不需要交互的 Layer 加 listening={false}

```tsx
<Layer listening={false}>
  ...
</Layer>
```

适合：

```text
背景网格
页面边界
静态辅助层
```

---

# 3. 使用 FastLayer

Konva 有 `FastLayer`，适合纯绘制、不交互。

例如：

```tsx
import { FastLayer } from "react-konva";
```

用于：

```text
背景网格
大量不可交互装饰
```

但是注意：

```text
FastLayer 不支持事件命中
```

所以只能给不可交互内容。

---

# 4. 对复杂节点 cache

对于复杂的 group，比如 AI 工作流节点，如果内部有很多：

```text
Rect
Text
Image
Line
阴影
圆角
```

可以考虑 `cache()`。

React-Konva 里通常用 ref：

```tsx
const groupRef = useRef<Konva.Group>(null);

useEffect(() => {
  groupRef.current?.cache();
  groupRef.current?.getLayer()?.batchDraw();
}, [/* 影响外观的依赖 */]);
```

但注意：

```text
动态变化很频繁的节点不适合 cache
```

适合：

```text
外观复杂但不常变化的 AI 节点
缩略图节点
静态装饰
```

---

# 5. 拖动时临时关闭阴影/滤镜

阴影和 blur 很耗性能。

可以在拖动时：

```text
dragging = true
```

然后节点渲染时：

```tsx
shadowBlur={isDragging ? 0 : 12}
```

或者：

```text
拖动中不显示滤镜
拖动结束恢复
```

这会大幅提升拖拽流畅度。

---

# 八、性能优化第六阶段：图片优化

画布卡顿很多时候是图片造成的。

---

## 1. 不要用超大原图直接绘制

如果图片原图是：

```text
4000 x 4000
```

但画布显示：

```text
500 x 500
```

仍然可能很耗内存。

推荐：

```text
导入图片时生成显示用缩略图
原图 URL 保留为 source
画布显示 thumbnailSrc
导出或 AI 时再用原图
```

数据结构可以后续加：

```ts
type ImageElement = {
  src: string;          // 原图
  previewSrc?: string;  // 画布显示用
}
```

短期不想改数据结构，可以先做：

```text
导入大图时 canvas resize 到最大 2048 或 1600
```

---

## 2. 图片组件缓存

对于图片元素：

```tsx
<Image image={imageObj} />
```

要确保不要每次 render 都重新创建 `HTMLImageElement`。

应该有图片缓存 hook，例如：

```ts
useImage(src)
```

并且 src 不变时 image 对象不变。

---

## 3. 滤镜慎用

Konva filters 比较重：

```text
brightness
contrast
saturation
blur
```

尤其是 blur。

优化策略：

```text
拖动/缩放时暂停 filter
拖动结束恢复
或者滤镜结果预烘焙成图片
```

---

# 九、性能优化第七阶段：MiniMap 降频

MiniMap 通常很容易偷偷拖慢画布。

---

## 问题

如果 MiniMap 订阅了：

```text
pages
selectedIds
zoom
pan
```

那么：

```text
拖动画布时 MiniMap 一直 render
拖动元素时 MiniMap 一直 render
选中变化时 MiniMap render
```

---

## 优化策略

### 1. MiniMap 用 React.memo

```tsx
export const MiniMap = React.memo(function MiniMap() {
  ...
});
```

---

### 2. MiniMap 低频刷新

例如：

```text
每 100ms 刷新一次
```

或者：

```text
dragging 时不刷新
dragEnd 后刷新
```

---

### 3. MiniMap 不画细节

MiniMap 只画：

```text
元素 bounding box
AI 节点矩形
视口框
```

不要画：

```text
图片内容
文字细节
滤镜
阴影
端口
复杂连线
```

# 十、性能优化第八阶段：WorkflowNodeView 优化

AI 工作流节点如果多了，会卡。

---

## 优化方向

```text
1. WorkflowNodeView 用 React.memo
2. 端口组件 WorkflowPort 用 React.memo
3. 节点内部静态文本 listening={false}
4. 只让节点外层 Group 响应拖拽
5. 节点预览图降采样
6. 节点运行状态变化只更新对应节点
7. 边线不需要点击时 listening={false}
```

---

## 避免这种情况

```text
一个节点 running 改变
所有节点全部 render
```

如果发生，说明订阅或 props 有问题。

# 十一、性能优化第九阶段：localStorage 和历史记录

虽然你已经做了手势历史合并，但仍要检查。

---

## 1. localStorage 保存不能跟着高频事件跑

自动保存应该监听真正需要保存的字段，并防抖。

危险情况：

```text
pan/zoom 每次变化都触发自动保存
workflowConnecting pointer 每次变化触发自动保存
marqueeSelecting 变化触发保存
```

优化方式：

```text
只保存 pages、activePageId、必要配置
pan/zoom 是否保存要谨慎
临时 UI 状态不要保存
```

---

## 2. 历史记录不要记录临时状态

不要把这些写进历史：

```text
拖拽中状态
框选中状态
临时连线 pointer
toolbarToast
contextMenu
running 中间态
```

历史只记录：

```text
元素最终位置
最终尺寸
最终节点 outputs
最终 edges
```

---

# 十二、性能优化第十阶段：CSS / DOM 层优化

除了 Konva，普通 HTML UI 也可能拖慢。

---

## 1. FloatingToolbar

浮动工具条如果在鼠标移动时持续计算：

```text

```

会卡。

优化：

```text

用 useMemo
拖动中可以隐藏 toolbar
拖动结束再显示
```

很多编辑器都是：

```text
拖动中隐藏浮动工具条
```

## 2. ContextMenu

右键菜单没问题，低频。

---

## 3. Sidebars

左侧图层列表如果元素很多，也可能卡。

优化：

```text
React.memo
虚拟列表
只显示当前页
拖拽时不实时刷新
```

# 十七、你最终想达到的状态

理想的画布交互应该是：

```text
拖动元素时：
只有被拖的 Konva 节点动，不让整个 React 树重渲染。

拖动结束时：
一次性写回 store。

缩放/平移时：
一帧最多更新一次 viewport。

背景/辅助线/装饰：
不参与事件命中。

MiniMap/Toolbar：
低频更新或拖动中隐藏。

图片：
显示用小图，导出/AI 用原图。
```

做到这些，画布会明显丝滑很多。
